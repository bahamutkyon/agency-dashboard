# Supabase 啟發的設計系統

> 類別：Backend & Data
> 開源的 Firebase 替代方案。深色翡翠綠主題，程式碼優先。

## 1. 視覺主題與氛圍

Supabase 的網站是一個原生於深色模式的開發者平台，散發著高階程式碼編輯器的美學氣質——以深邃的黑色背景（`#0f0f0f`、`#171717`）搭配翡翠綠點綴（`#3ecf8e`、`#00c573`），呼應品牌開源、PostgreSQL 綠的視覺識別。整個設計系統彷彿誕生於終端機視窗，再進化成精緻的行銷介面，卻絲毫沒有失去開發者的靈魂。

字型建立在「Circular」之上——一款字尾帶圓潤收筆的幾何無襯線字，柔化了技術感的銳利邊緣。在 72px 字級搭配 1.00 行高下，主視覺文案被壓縮到極致的垂直空間，形成密實而有力的宣言，毫無浪費。等寬字（Source Code Pro）則節制使用，僅出現在大寫的技術標籤上並帶 1.2px 字距，營造出「開發者控制台」的標記，串連起行銷網站與產品實際體驗。

讓 Supabase 與眾不同的是它精緻的 HSL 配色 token 系統。Supabase 幾乎所有顏色都使用帶 alpha 通道的 HSL 而非扁平的 hex 值（`--colors-crimson4`、`--colors-purple5`、`--colors-slateA12`），形成細膩的疊層系統，讓顏色透過透明度互相作用。深度便來自於這種半透明——`rgba(46, 46, 46)` 的邊框、`rgba(41, 41, 41, 0.84)` 的表面、半透明的點綴——這些元素全部與深色背景融合，從最少的色彩素材中調出豐富而有層次的色盤。

翡翠綠（`#3ecf8e`）只在特定場合出現——Supabase logo、連結色（`#00c573`）、邊框高光（`rgba(62, 207, 142, 0.3)`）——永遠作為「這是 Supabase」的信號，而非裝飾元素。主要 CTA 採用藥丸形按鈕（9999px 圓角），與次要元素的標準 6px 圓角形成對比，建立清晰的視覺層級。

**核心特徵：**
- 原生深色模式：近黑背景（`#0f0f0f`、`#171717`）——絕不使用純黑
- 翡翠綠品牌點綴（`#3ecf8e`、`#00c573`）節制使用，作為識別標記
- Circular 字型——幾何無襯線，字尾圓潤
- Source Code Pro 用於大寫技術標籤（1.2px 字距）
- 基於 HSL 的色彩 token 系統，搭配 alpha 通道做半透明疊層
- 主要 CTA 採用藥丸按鈕（9999px），次要元素 6px 圓角
- 中性灰階從 `#171717` 經 `#898989` 到 `#fafafa`
- 邊框系統使用深灰（`#2e2e2e`、`#363636`、`#393939`）
- 極少使用陰影——深度來自邊框對比與透明度
- Radix 顏色基本色（crimson、purple、violet、indigo、yellow、tomato、orange、slate）

## 2. 配色系統與角色

### 品牌色
- **Supabase Green** (`#3ecf8e`)：主要品牌色、logo、點綴邊框
- **Green Link** (`#00c573`)：互動綠，用於連結與動作
- **Green Border** (`rgba(62, 207, 142, 0.3)`)：細緻的綠色邊框點綴

### 中性階（深色模式）
- **Near Black** (`#0f0f0f`)：主按鈕背景，最深層表面
- **Dark** (`#171717`)：頁面背景，主畫布
- **Dark Border** (`#242424`)：水平分隔線、區段分隔
- **Border Dark** (`#2e2e2e`)：卡片邊框、頁籤邊框
- **Mid Border** (`#363636`)：按鈕邊框、分隔線
- **Border Light** (`#393939`)：次級邊框
- **Charcoal** (`#434343`)：第三層邊框、深色點綴
- **Dark Gray** (`#4d4d4d`)：較深的次要文字
- **Mid Gray** (`#898989`)：弱化文字、連結色
- **Light Gray** (`#b4b4b4`)：次級連結文字
- **Near White** (`#efefef`)：淺色邊框、細膩表面
- **Off White** (`#fafafa`)：主要文字、按鈕文字

### Radix 顏色 token（HSL）
- **Slate Scale**：`--colors-slate5` 到 `--colors-slateA12`——中性漸進
- **Purple**：`--colors-purple4`、`--colors-purple5`、`--colors-purpleA7`——點綴光譜
- **Violet**：`--colors-violet10` (`hsl(251, 63.2%, 63.2%)`)——鮮明點綴
- **Crimson**：`--colors-crimson4`、`--colors-crimsonA9`——暖色點綴／警示
- **Indigo**：`--colors-indigoA2`——細膩的藍色暈染
- **Yellow**：`--colors-yellowA7`——提醒／警告
- **Tomato**：`--colors-tomatoA4`——錯誤點綴
- **Orange**：`--colors-orange6`——暖色點綴

### 表面與覆蓋層
- **Glass Dark** (`rgba(41, 41, 41, 0.84)`)：半透明暗色覆蓋層
- **Slate Alpha** (`hsla(210, 87.8%, 16.1%, 0.031)`)：極淡的藍色暈染
- **Fixed Scale Alpha** (`hsla(200, 90.3%, 93.4%, 0.109)`)：淺色霧面覆蓋層

### 陰影
- Supabase 在深色主題中**幾乎不使用陰影**。深度來自邊框對比與表面色差，而非 box-shadow。聚焦狀態使用 `rgba(0, 0, 0, 0.1) 0px 4px 12px`——極簡、實用。

## 3. 字型系統

### 字型家族
- **主要字型**：`Circular`，備援：`custom-font, Helvetica Neue, Helvetica, Arial`
- **等寬字型**：`Source Code Pro`，備援：`Office Code Pro, Menlo`

### 階層

| 角色 | 字型 | 字級 | 字重 | 行高 | 字距 | 備註 |
|------|------|------|------|------|------|------|
| Display Hero | Circular | 72px (4.50rem) | 400 | 1.00（緊） | normal | 極致密度，零浪費 |
| Section Heading | Circular | 36px (2.25rem) | 400 | 1.25（緊） | normal | 功能區段標題 |
| Card Title | Circular | 24px (1.50rem) | 400 | 1.33 | -0.16px | 略帶負字距 |
| Sub-heading | Circular | 18px (1.13rem) | 400 | 1.56 | normal | 次級標題 |
| Body | Circular | 16px (1.00rem) | 400 | 1.50 | normal | 標準內文 |
| Nav Link | Circular | 14px (0.88rem) | 500 | 1.00–1.43 | normal | 導覽項目 |
| Button | Circular | 14px (0.88rem) | 500 | 1.14（緊） | normal | 按鈕標籤 |
| Caption | Circular | 14px (0.88rem) | 400–500 | 1.43 | normal | 中繼資料、標籤 |
| Small | Circular | 12px (0.75rem) | 400 | 1.33 | normal | 小字、頁腳連結 |
| Code Label | Source Code Pro | 12px (0.75rem) | 400 | 1.33 | 1.2px | `text-transform: uppercase` |

### 原則
- **字重節制**：幾乎所有文字都使用字重 400（regular/book）。字重 500 只出現在導覽連結與按鈕標籤上。系統中沒有粗體（700）——層級透過字級而非字重來建立。
- **1.00 主視覺行高**：主視覺文字被壓縮到絕對零的行距。這是定義性的字型手法——文字像終端機指令一樣密實、高效、毫不浪費垂直空間。
- **卡片的負字距**：卡片標題使用 -0.16px 字距，是細膩的緊縮，讓它與內文區分開來卻不顯刻意。
- **等寬字作為儀式**：Source Code Pro 大寫並帶 1.2px 字距是「開發者控制台」的語氣——節制使用於連結到產品體驗的技術標籤。
- **幾何感的個性**：Circular 圓潤的字尾為冰冷的技術介面注入溫度。這款字是人性化的元素。

## 4. 元件樣式

### 按鈕

**主要藥丸（深色）**
- 背景：`#0f0f0f`
- 文字：`#fafafa`
- Padding：8px 32px
- 圓角：9999px（完整藥丸）
- 邊框：`1px solid #fafafa`（深色上的白色邊框）
- Focus 陰影：`rgba(0, 0, 0, 0.1) 0px 4px 12px`
- 用途：主要 CTA（「Start your project」）

**次要藥丸（深色，弱化）**
- 背景：`#0f0f0f`
- 文字：`#fafafa`
- Padding：8px 32px
- 圓角：9999px
- 邊框：`1px solid #2e2e2e`（深色邊框）
- 不透明度：0.8
- 用途：搭配主要按鈕的次級 CTA

**Ghost 按鈕**
- 背景：透明
- 文字：`#fafafa`
- Padding：8px
- 圓角：6px
- 邊框：`1px solid transparent`
- 用途：第三層動作、圖示按鈕

### 卡片與容器
- 背景：深色表面（`#171717` 或稍淺）
- 邊框：`1px solid #2e2e2e` 或 `#363636`
- 圓角：8px–16px
- 無可見陰影——靠邊框定義邊緣
- 內距：16px–24px

### 頁籤
- 邊框：`1px solid #2e2e2e`
- 圓角：9999px（藥丸頁籤）
- 啟用：綠色點綴或更淺的表面
- 未啟用：深色、弱化

### 連結
- **Green**：`#00c573`——Supabase 品牌連結
- **Primary Light**：`#fafafa`——深色背景上的標準連結
- **Secondary**：`#b4b4b4`——弱化連結
- **Muted**：`#898989`——第三層連結、頁腳

### 導覽
- 與頁面相同的深色背景（`#171717`）
- 帶綠色圖示的 Supabase logo
- Circular 14px 字重 500 用於導覽連結
- 簡潔的水平佈局，搭配產品下拉選單
- 綠色「Start your project」CTA 藥丸按鈕
- 黏性 header 行為

## 5. 間距與佈局

### 間距系統
- 基本單位：8px
- 尺度：1px、4px、6px、8px、12px、16px、20px、24px、32px、40px、48px、90px、96px、128px
- 值得注意的大幅跳階：48px → 90px → 96px → 128px 用於主要區段間距

### 網格與容器
- 內容置中並有寬裕的 max-width
- 全寬深色區段搭配受限的內部內容
- 功能格線：以圖示為主、卡片尺寸一致的格線
- 「Trusted by」區段的 logo 格線
- 頁腳：深色背景上的多欄式

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|------|----------|
| Mobile | <600px | 單欄、堆疊佈局 |
| Desktop | >600px | 多欄格線、展開佈局 |

*註：Supabase 採用相當精簡的斷點系統——主要只有單一 600px 斷點，反映出 mobile-first 漸進增強的取向。*

### 留白哲學
- **戲劇化的區段間距**：主要區段之間 90px–128px 的間距創造出電影般的節奏感——每個區段都像在黑色虛空中的獨立場景。
- **密實的內容區塊**：在區段內部，間距很緊湊（16px–24px），形成集中的資訊群集。
- **以邊框定義空間**：Supabase 不靠「留白＋陰影」做分隔，而是在深色背景上使用細邊框——以線條而非間隙來區分。

### 圓角尺度
- 標準（6px）：Ghost 按鈕、小元素
- 舒適（8px）：卡片、容器
- 中等（11px–12px）：中型面板
- 大型（16px）：功能卡片、主要容器
- 藥丸（9999px）：主要按鈕、頁籤指示器

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|------|----------|------|
| 平面（Level 0） | 無陰影，邊框 `#2e2e2e` | 預設狀態，大多數表面 |
| 細邊框（Level 1） | 邊框 `#363636` 或 `#393939` | 互動元素、hover |
| 聚焦（Level 2） | `rgba(0, 0, 0, 0.1) 0px 4px 12px` | 僅 focus 狀態 |
| 綠色點綴（Level 3） | 邊框 `rgba(62, 207, 142, 0.3)` | 品牌強調元素 |

**陰影哲學**：Supabase 刻意避開陰影。在原生深色模式設計中，陰影幾乎看不見，沒有實際作用。深度反而透過精緻的邊框層級來傳達——從 `#242424`（幾乎不可見）經 `#2e2e2e`（標準）到 `#393939`（突出）。30% 不透明度的綠色點綴邊框（`rgba(62, 207, 142, 0.3)`）就是「升起」狀態——品牌色本身成為深度的信號。

## 7. Do's and Don'ts

### Do
- 使用近黑背景（`#0f0f0f`、`#171717`）——深度來自灰色邊框層級
- 節制使用 Supabase 綠（`#3ecf8e`、`#00c573`）——它是識別標記，不是裝飾
- 幾乎所有元素都用字重 400 的 Circular——500 只給按鈕與導覽
- 主視覺文字設定 1.00 行高——零行距是字型簽名
- 透過邊框色差（`#242424` → `#2e2e2e` → `#363636`）營造深度
- 藥丸形（9999px）只用於主要 CTA 與頁籤
- 採用 HSL 加 alpha 的顏色做半透明疊層效果
- 用 Source Code Pro 大寫標籤作為開發者情境的標記

### Don't
- 不要加 box-shadow——在深色背景上看不見，會破壞以邊框定義深度的系統
- 不要用粗體（700）字重——系統只用 400 與 500
- 不要把綠色放在背景或大面積表面——它是給邊框、連結、小點綴用的
- 不要把暖色（crimson、orange）當主要設計元素——它們只是狀態的語意 token
- 不要把主視覺行高調高於 1.00——這份密度是刻意的
- 不要在按鈕用大圓角（16px 以上）——藥丸（9999px）或標準（6px），中間沒有
- 不要把主要表面背景調亮於 `#171717`——黑暗是結構性的
- 別忘了半透明邊框——`rgba` 邊框色就是疊層機制

## 8. 響應式行為

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|------|----------|
| Mobile | <600px | 單欄、功能堆疊、精簡導覽 |
| Desktop | >600px | 多欄格線、完整導覽、展開區段 |

### 摺疊策略
- Hero：72px → 等比縮小
- 功能格線：多欄 → 單欄堆疊
- Logo 列：水平 → 換行格線
- 導覽：完整 → 漢堡選單
- 區段間距：90–128px → 48–64px
- 按鈕：行內 → 全寬堆疊

## 9. Agent Prompt 指南

### 快速色彩參考
- 背景：`#0f0f0f`（按鈕）、`#171717`（頁面）
- 文字：`#fafafa`（主要）、`#b4b4b4`（次要）、`#898989`（弱化）
- 品牌綠：`#3ecf8e`（品牌）、`#00c573`（連結）
- 邊框：`#242424`（細膩）、`#2e2e2e`（標準）、`#363636`（突出）
- 綠色邊框：`rgba(62, 207, 142, 0.3)`（點綴）

### 範例元件 Prompt
- 「在 #171717 背景上建立 hero 區段。標題用 72px Circular 字重 400、行高 1.00、`#fafafa` 文字。副標 16px Circular 字重 400、行高 1.50、`#b4b4b4`。藥丸 CTA 按鈕（`#0f0f0f` 底、`#fafafa` 文字、9999px 圓角、8px 32px padding、`1px solid #fafafa` 邊框）。」
- 「設計一張功能卡片：`#171717` 背景、`1px solid #2e2e2e` 邊框、16px 圓角。標題 24px Circular 字重 400、字距 -0.16px。內文 14px 字重 400、`#898989` 文字。」
- 「打造導覽列：`#171717` 背景。Circular 14px 字重 500、`#fafafa` 文字。Supabase logo 加綠色圖示靠左對齊。綠色藥丸 CTA「Start your project」靠右對齊。」
- 「建立技術標籤：Source Code Pro 12px、大寫、字距 1.2px、`#898989` 文字。」
- 「設計框架 logo 格線：深色上 6 欄佈局、灰階 logo 60% 不透明度、區段之間 `1px solid #2e2e2e`。」

### 迭代指南
1. 從 `#171717` 背景開始——一切都是原生深色模式
2. 綠色是品牌識別標記——只用於連結、logo、點綴邊框
3. 深度來自邊框（`#242424` → `#2e2e2e` → `#363636`），不是陰影
4. 字重 400 是預設——500 只給互動元素
5. 主視覺行高 1.00 是字型簽名手法
6. 主要動作用藥丸（9999px）、次要用 6px、卡片用 8–16px
7. 帶 alpha 通道的 HSL 創造出精緻的半透明疊層
