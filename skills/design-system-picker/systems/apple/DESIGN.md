# 受 Apple 啟發的設計系統

> 類別：媒體與消費端
> 消費電子。高級留白、SF Pro、電影感影像。

## 1. 視覺主題與氛圍

Apple 的網頁語彙是一套精準的編輯系統，在畫廊般的沉靜與零售般的密集資訊區塊之間交替。整體視覺基調保持克制：寬廣的中性畫布、低調的介面外殼，將幾乎所有表現力的重量都交給產品影像。介面被刻意地工程化讓自身隱形，好讓硬體、材質與外觀選項成為敘事主角。

在分析的五個頁面之間，節奏一致但並不單調。行銷頁面（首頁與 Environment）採電影感的黑白章節切分；商業頁面（Store 與購買流程）則導入更緊湊的間距、更多功能控制與更密集的卡片堆疊，卻不破壞核心品牌語法。最終呈現出一個系統、兩種檔位：展示模式與交易模式。

字型是穩定整體的關鍵。SF Pro Display 以緊湊行高與受控字距承擔主視覺與商品標題的階層，SF Pro Text 則處理產品後設資料、導覽、篩選器與密集的選項介面。字型整體保持低調，但尺寸範圍足夠寬廣，能同時支援廣告看板般的主視覺訊息與微型工具標籤。

**主要特徵：**
- 二元區塊節奏：深黑場景（`#000000`）與淺中性場域（`#f5f5f7`）交替
- 單一藍色家族承擔行動與連結語意（`#0071e3`、`#0066cc`、`#2997ff`）
- 一套系統內的雙運作模式：電影感展示模組與密集商業配置器
- 高度仰賴影像與材質呈現；介面外殼在視覺上保持纖薄
- 緊湊的標題度量（SF Pro Display、semibold）搭配緊湊的內文／連結字型（SF Pro Text）
- 藥丸與膠囊幾何作為標誌性行動語彙（`18px` 至 `980px` 與圓形控制元件）
- 深度節制地使用；對比與表面區隔承擔大部分層次工作
- 多頁面色塊節奏：黑色主視覺章節 -> 淺中性商品展示場域 -> 工具白色零售表面 -> 深色微型控制表面

## 2. 配色系統與角色

> **來源頁面：** `https://www.apple.com/`、`https://www.apple.com/environment/`、`https://www.apple.com/store`、`https://www.apple.com/shop/buy-iphone/iphone-17-pro`、`https://www.apple.com/shop/accessories/all`

### 主色
- **Absolute Black**（`#000000`）：沉浸式主視覺畫布、高張力產品章節、深色介面錨點。
- **Pale Apple Gray**（`#f5f5f7`）：特色帶狀區、比較區塊與編輯轉場的主要淺色表面。
- **Near-Black Ink**（`#1d1d1f`）：淺色畫布上的主要文字與深色填色控制元件。

### 次要色與強調色
- **Apple Action Blue**（`#0071e3`）：主要行動填色與焦點訊號的品牌強調色。
- **Body Link Blue**（`#0066cc`）：為長文閱讀優化的內嵌連結色。
- **High-Luminance Link Blue**（`#2997ff`）：深色場景中需要更強對比時使用的明亮連結色。

### 表面與背景
- **Pure White Canvas**（`#ffffff`）：零售／產品列表背景與密集交易區塊。
- **Graphite Surface A**（`#272729`）：深色卡片與媒體控制情境層。
- **Graphite Surface B**（`#262629`）：控制群組稍深的深色工具層。
- **Graphite Surface C**（`#28282b`）：浮起的深色輔助表面。
- **Graphite Surface D**（`#2a2a2c`）：較豐富深色場景中用於分層的最深浮起階。

### 中性色與文字
- **Secondary Neutral Gray**（`#6e6e73`）：內文次要文案、輔助說明、三級後設資料。
- **Soft Border Gray**（`#d2d2d7`）：分隔線、細微外框與低調的工具容器。
- **Mid Border Gray**（`#86868b`）：產品配置與篩選情境中較強的欄位外框。
- **Utility Dark Gray**（`#424245`）：商店情境中介於深色文字與深色表面之間的中介色。

### 語意色與強調色
- **Selection/Focus Signal**（`#0071e3`）：跨行銷與商業情境共用的焦點與選取狀態訊號。
- **Error/Warning/Success**：在擷取的頁面集合中未見一致出現的明確語意色盤。

### 漸層系統
- 擷取的頁面壓倒性地以實色表面驅動。視覺豐富度來自攝影與材質渲染，而非介面上持續存在的漸層。

- **用法切分：** 展示家族承擔主視覺／產品標題與商品標頭；內文家族處理導覽、控制元件、標籤與密集商業文案。

### 原則
- **跨頁型的連貫性：** 同一套字型 DNA 橫跨電影感發表會與產品購買流程，避免行銷與商業之間出現品牌斷裂。
- **大尺度下的壓縮：** 展示階層採緊密行距與受控字距，呈現出機械感且以產品為主的氛圍。
- **零售深度下的可讀密度：** SF Pro Text 在緊湊性與垂直節奏之間取得平衡，足以承載長產品列表與選項矩陣。
- **度量得宜的字重階梯：** 600 是主要的強調字重；700 僅在特定情境出現；300 則用於大尺寸文字的對比，使用節制。

### 字型替代備註
- 最接近的免費替代字型：在文字密集的場景使用 `Inter`，標題則以 `Inter Tight` 近似 SF Pro Display 的度量。
- 替換時，將內文尺寸的行高略微增加（+0.02 至 +0.06），並降低負字距強度，以保留可讀性。

## 4. 元件樣式

### 按鈕
- **主要填色行動：** `#0071e3` 背景、`#ffffff` 文字、8px 圓角、緊湊的水平內距（常見 8px 15px）。用於決定性的購買／進行動作。
- **深色填色行動：** `#1d1d1f` 背景、`#ffffff` 文字、8px 圓角。當淺色表面需要克制的高對比主要按鈕時使用。
- **藥丸／膠囊行動家族：** 大型膠囊行動圓角介於 `18px`-`56px`，極端藥丸連結則為 `980px`。確立 Apple 柔和卻精準的 CTA 輪廓。
- **工具篩選／按鈕外殼：** 淺色外殼（`#fafafc` 或半透明白）搭配細微灰色邊框（`#d2d2d7` / `#86868b`），用於密集配置情境。
- **按下行為：** 啟用中的控制元件通常會微縮 scale 或略微改變填色，回饋實體按下的確認感。

### 卡片與容器
- **編輯／產品卡片：** 落在 `#f5f5f7` 或白色場域的淺色卡片，框邊極簡、以圖像優先的構圖。
- **深色工具卡片：** 石墨階（`#272729` 至 `#2a2a2c`）用於覆層、媒體控制與深色情境模組。
- **配置器面板：** 圓角容器（常見 12px-18px），邊框定義明確但克制。
- **輪播／聚焦模組：** 較大圓角的外殼（`28px`-`36px`）用於特色內容跑道。

### 輸入框與表單
- **零售輸入欄位：** 半透明或白色背景、深色文字（`#1d1d1f`）、以邊框為主的容器（`#86868b`）。
- **選擇控制元件：** 在產品選擇介面中常見圓形／切換式的控制幾何。
- **密度策略：** 表單欄位在視覺上保持低調，好讓裝置影像與價格階層保持主導。

### 導覽
- **全域行銷導覽：** 緊湊的深色半透明列、小型文字連結、克制的圖示語彙。
- **商店／子商店導覽層：** 額外的工具列、晶片元件與分段控制，用於分類與產品篩選。
- **連結階層：** 連結藍仍是主要互動訊號，中性文字則支撐密集的導覽集合。

### 圖像處理
- **物件優先的攝影：** 硬體與配件被擺放在受控的實色表面之前作為主角。
- **高保真的材質呈現：** 反射與材質細節是視覺說服力的核心。
- **混合構圖：** 滿版主視覺場景與圓角零售卡片、緊湊裁切的商品縮圖並存。

### 其他特色元件
- **產品配置器矩陣：** 選項堆疊與選擇器，結合晶片元件、廣播式控制與情境定價／摘要區塊。
- **輪播控制點與箭頭：** 在低調覆層中使用圓形控制語彙，承擔展示廊的瀏覽。
- **Environment 故事面板：** 將編輯感字型與電影感產品／環境影像結合的敘事章節。

## 5. 間距與佈局

### 間距系統
- 基本單位實際上是 `8px`，但系統支援密集的微步進以做到精準對齊。
- 跨頁面常用的間距值：`2`、`4`、`6`、`7`、`8`、`9`、`10`、`12`、`14`、`17`、`20` px。
- 跨行銷與零售流程都可見的通用節奏常數：`8px` 單位骨架，搭配 `14-20px` 工具間距用於元件內距與列表間距。

### 網格與容器
- **展示頁面：** 大型中央欄位、寬廣的水平呼吸空間、滿版色塊章節。
- **商業頁面：** 較緊湊的多欄產品與控制網格，常見模組化堆疊。
- **容器行為：** 受限的可讀核心，在桌面寬度搭配充裕的外緣邊距。

### 留白哲學
- **場景節奏：** 主要視覺章節採寬廣的上下呼吸空間。
- **必要處的資訊壓縮：** 零售頁面刻意壓縮間距，以在單一視窗內呈現更多可操作資訊。
- **以對比帶動分隔：** 區段轉場仰賴表面變化而非裝飾性分隔。

### 圓角尺度
- **5px：** 微型工具連結／標籤與小型外殼。
- **8px-12px：** 標準控制元件與緊湊欄位。
- **16px-18px：** 卡片、模組框架與商業面板。
- **28px-36px：** 較大模組與聚焦容器。
- **56px / 100px / 980px：** 膠囊、大型藥丸與標誌性拉長 CTA 形狀。
- **50%：** 圓形媒體與選擇控制元件。

## 6. 深度與層級

| 層級 | 處理方式 | 用途 |
|------|-----------|-----|
| Level 0 | 扁平中性表面（`#ffffff`、`#f5f5f7`、`#000000`） | 主要敘事與產品舞台 |
| Level 1 | 細微邊框容器（`#d2d2d7`、`#86868b`） | 篩選器、輸入欄位、工具卡片 |
| Level 2 | 柔和陰影（如存在則為 `rgba(0,0,0,0.08)` 至 `rgba(0,0,0,0.22)`） | 高亮卡片與浮起商品模組 |
| Level 3 | 深色表面分層（`#272729` -> `#2a2a2c`） | 覆層、媒體控制、深色工具群組 |
| 無障礙 | 藍色焦點訊號（`#0071e3`） | 鍵盤與選取強調 |

深度刻意保持節制。Apple 偏好色調對比、表面分層與構圖階層，而非堆疊厚重的陰影。

### 裝飾性深度
- 裝飾性深度主要由攝影的真實感與材質渲染所建立，並非由合成介面效果產生。
- 半透明覆層與玻璃感工具列在導覽與控制元件上提供輕微的氛圍分層。

## 7. 使用準則

### 應該做
- 以中性三色（`#000000`、`#f5f5f7`、`#ffffff`）作為結構基底。
- 將藍色強調色保留給真正的行動與導覽語意。
- 保持字型緊湊而精心安排，特別是在展示尺度上。
- 在控制元件與關鍵行動上維持膠囊／圓形的幾何語彙。
- 讓產品影像承擔視覺戲劇性，介面外殼保持低調。
- 在密集零售情境中使用邊框定義的容器，而非過度裝飾的卡片。
- 在展示模組與交易模組之間保持清楚的區隔，但讓核心 token 共用。

### 不要做
- 不要引入會與 Apple 藍競爭的廣泛次要強調色盤。
- 不要在核心介面外殼中過度使用陰影、發光效果或裝飾漸層。
- 不要混用不相干的字型家族或任意放鬆字距。
- 不要將所有圓角統一為單一數值；Apple 採目的明確的圓角階層。
- 不要在商業模組上加上過厚的邊框或聒噪的視覺效果。
- 不要拿掉深色與淺色章節之間的中性對比節奏。
- 不要把行銷與購買流程當作各自獨立的設計系統。

## 8. 響應式行為

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|-------|-------------|
| Small Mobile | 374px 及以下 | 緊湊零售控制、單欄產品堆疊 |
| Mobile | 375px-640px | 單欄模組、緊湊的行動列、收斂的選擇器 |
| Tablet | 641px-833px | 擴張的卡片，1-2 欄之間的混合轉換 |
| Tablet Wide | 834px-1023px | 更穩定的多欄商品展示、較大的文字區塊 |
| Desktop | 1024px-1240px | 完整零售佈局與產品比較結構 |
| Desktop Wide | 1241px-1440px | 行銷主視覺擴張與更寬廣的區段間距 |
| Large Desktop | 1441px+ | 最大化的章節呼吸空間與寬幅編輯構圖 |

### 觸控目標
- 主要與次要行動通常以易於點擊的藥丸／按鈕幾何呈現。
- 圓形媒體與選擇控制元件在行動情境中對齊最小可觸碰意圖。
- 密集的商業介面使用緊湊標籤，但透過外圍形狀內距維持清楚的點擊區域。

### 收合策略
- 行銷主視覺字型以離散階層縮放，同時保留階層對比。
- 產品與商業網格從多欄收合為堆疊卡片，並持續呈現選擇器。
- 工具導覽壓縮為更簡單的連結／控制群組，同時保留關鍵行動。
- 選項／配置群組變為垂直排列，讓購買流程在小螢幕上保持線性。

### 圖像行為
- 產品影像在各斷點之間保留比例與置中性。
- 行動裝置上主視覺仍占主導地位，文字會依據媒體優先重新配置。
- 零售縮圖透過更緊湊的裁切邏輯與更密集的卡片堆疊保持可讀。
- 隨著佈局密度增加，圖像主導模組仍持續錨定畫面節奏。

### 迭代指南
1. 在調整強調色之前，先鎖定中性基底（`#000000`、`#f5f5f7`、`#ffffff`）。
2. 將藍色強調色保留得稀少且目的明確；若全是藍色，階層就會崩塌。
3. 按以下順序調整字型：展示尺度、內文可讀性、再到微型標籤。
4. 依元件類別（欄位、卡片、膠囊、圓形）對應圓角，而非一體適用單一圓角。
5. 從展示區塊進到商業區塊時，逐步提高密度。
6. 每次修改後，確認產品影像仍是畫面中最強的視覺層。

### 已知的缺漏
- 在擷取的頁面集合中，未一致出現明確的語意狀態色（錯誤／警告／成功）。
- 部分互動微狀態因模組而異，無法以通用系統 token 表達。
- 少數零售模組顯露出情境專屬的字型覆寫，並未跨全部五個頁面出現。
