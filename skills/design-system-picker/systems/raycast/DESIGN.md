# 受 Raycast 啟發的設計系統

> 類別：Developer Tools
> 生產力啟動器。俐落的深色金屬感，搭配鮮明的漸層強調。

## 1. 視覺主題與氛圍

Raycast 的行銷網站給人的感受，就像在窺視一台精密儀器的暗色內裝 — 一只用黑曜石雕成的瑞士手錶機芯。背景不只是「暗」，而是一種帶著藍色冷調的近黑色（`#07080a`），整體呈現出彷彿身處 macOS 原生應用程式內部，而非一個網頁的氛圍。每一個表面、每一道邊框、每一道陰影，都經過精心校準，喚起一種高效能桌面工具的感覺：快速、極簡、值得信任。

最具標誌性的手法是借鑑 macOS 視窗外觀的多層陰影系統：多層 box-shadow 加上內陰影高光，模擬出實體深度，讓卡片與按鈕看起來就像放在深色桌面上、實際被按壓或浮起的玻璃元件。再搭配 Raycast Red（`#FF6363`） — 幾乎只用在 hero 區塊那道標誌性的斜紋圖樣上 — 整套配色讓品牌讀起來是「有個性的強力工具」。紅色不會主宰畫面，它只是用來點睛。

Inter 字型貫穿整站 — 標題、內文、按鈕、說明文字都是它，並大量使用 OpenType 功能（`calt`、`kern`、`liga`、`ss03`），形成一致且可讀的字體語彙。內文採用正字距（0.2px–0.4px），這在深色 UI 中相當少見，但反而讓文字呈現一種透氣、輕盈的質感，平衡了那些密實的深色表面。GeistMono 則出現在程式碼元素上，強化開發者工具的身份。

**主要特徵：**
- 近黑帶藍色冷調的背景（`#07080a`）— 不是純黑，而是微微偏藍
- 仿 macOS 原生的陰影系統，多層內陰影高光營造實體深度
- Raycast Red（`#FF6363`）作為點睛色 — 出現在 hero 斜紋，而非到處鋪滿
- Inter 配上正字距（0.2px），讓深色模式也能透氣易讀
- 由 Radix UI 元件原語驅動互動層
- 微弱的白色半透明邊框（不透明度 0.06–0.1），用來在深色表面上界定範圍
- 鍵盤快捷鍵採用漸層鍵帽與厚重陰影樣式

## 2. 配色系統與角色

- **Near-Black Blue**（`#07080a`）：主要頁面背景 — 微微偏藍冷的基礎暗空間

- **Raycast Red**（`#FF6363` / `hsl(0, 100%, 69%)`）：品牌強調色 — hero 斜紋、危險狀態、關鍵重點

### 次要與強調

### 表面與背景

### 漸層系統
- **Keyboard Key Gradient**：由 `#121212`（上）漸層到 `#0d0d0d`（下） — 模擬實體按鍵深度
- **Warm Glow**：`rgba(215, 201, 175, 0.05)` 放射狀擴散 — 在特色元素後方營造低調的暖色環境光

## 3. 字型系統

### 字型家族
- **Primary**：`Inter` — 人文無襯線字型，全站通用。Fallback：`Inter Fallback`、系統無襯線字型
- **System**：`SF Pro Text` — Apple 系統字型，用於少數 macOS 原生 UI 元件。Fallback：`SF Pro Icons`、`Inter`、`Inter Fallback`
- **Monospace**：`GeistMono` — Vercel 的等寬字型，用於程式碼元素。Fallback：`ui-monospace`、`SFMono-Regular`、`Roboto Mono`、`Menlo`、`Monaco`
- **OpenType 功能**：全域啟用 `calt`、`kern`、`liga`、`ss03`；display 文字另啟用 `ss02`、`ss08`；hero 標題則停用 ligature（`"liga" 0`）

### 原則
- **深色背景仍採正字距**：與多數深色 UI 採用緊湊或中性字距不同，Raycast 給內文 +0.2px 到 +0.4px 的正字距，創造一種透氣、易讀的感受，補償深色背景帶來的壓迫感
- **字重 500 為基準**：大部分內文使用中等字重（500）而非 regular（400） — 多一點點重量明顯提升深色表面的可讀性
- **Display 克制**：hero 文字採 64px／600，自信但不過大 — Raycast 拒絕字型上的炫技，偏好功能性的優雅
- **OpenType 全面啟用**：`ss03`（stylistic set 3）在所有 Inter 文字上全域啟用，讓字型多了一點幾何感與工具氣質

## 4. 元件與模式

### 按鈕
- **Primary Pill**：透明背景，白色文字，藥丸形（86px 圓角），多層內陰影（`rgba(255, 255, 255, 0.1) 0px 1px 0px 0px inset`）。Hover：opacity 0.6
- **Secondary Button**：透明背景，白色文字，6px 圓角，`1px solid rgba(255, 255, 255, 0.1)` 邊框，細微的 drop shadow（`rgba(0, 0, 0, 0.03) 0px 7px 3px`）。Hover：opacity 0.6
- **Ghost Button**：無背景、無邊框，灰色文字（`#6a6b6c`），86px 圓角，同樣的內陰影。Hover：opacity 0.6，文字提亮為白
- **CTA（Download）**：半透明白底（`hsla(0, 0%, 100%, 0.815)`），深色文字（`#18191a`），藥丸形。Hover：底色提升為完全不透明的白（`hsl(0, 0%, 100%)`）
- **過場**：所有按鈕的 hover 都以 opacity 過場處理，而不是切換背景色 — 這是 Raycast 標誌性的互動模式

### 卡片與容器
- **Standard Card**：`#101111` 表面，`1px solid rgba(255, 255, 255, 0.06)` 邊框，12px–16px 圓角
- **Elevated Card**：環形陰影外圈 `rgb(27, 28, 30) 0px 0px 0px 1px` 加上內圈 `rgb(7, 8, 10) 0px 0px 0px 1px inset` — 形成雙環包覆
- **Feature Card**：16px–20px 圓角，背後加上微弱的暖光（`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px`）
- **Hover**：卡片透過提高邊框不透明度或加強陰影微微亮起

### 輸入框與表單
- 深色輸入欄位：`#07080a` 背景、`1px solid rgba(255, 255, 255, 0.08)` 邊框、8px 圓角
- Focus 狀態：邊框提亮，浮現藍色光暈環（`hsla(202, 100%, 67%, 0.15)`）
- 文字：輸入文字 `#f9f9f9`，placeholder `#6a6b6c`
- 標籤：`#9c9c9d`，14px 字重 500

### 導覽
- **頂部導覽**：深色背景與頁面融合，白色文字連結，16px 字重 500
- **導覽連結**：灰色文字（`#9c9c9d`）→ hover 時轉白並出現底線
- **CTA 按鈕**：導覽列右端的半透明白色藥丸鈕
- **行動裝置**：收合為漢堡選單，維持深色主題
- **Sticky**：導覽固定於頂部，下緣有細微邊框分隔

### 圖像處理
- **產品截圖**：仿 macOS 視窗外觀 — 圓角（12px）、深陰影模擬懸浮視窗
- **滿版區塊**：深色截圖能無縫融入深色背景
- **Hero 圖像**：Raycast Red 的斜紋圖樣 — 抽象、幾何、定義品牌
- **App UI 嵌入**：直接展示 Raycast 命令選單與擴充套件 — 產品即內容

### 鍵盤快捷鍵按鍵
- **鍵帽樣式**：漸層背景（`#121212` → `#0d0d0d`），厚重的多層陰影（`rgba(0, 0, 0, 0.4) 0px 1.5px 0.5px 2.5px` 加內陰影），創造寫實的實體按鍵感
- 圓角：個別鍵帽為 4px–6px

### 徽章與 Tag
- **中性徽章**：`#1b1c1e` 背景，白色文字，6px 圓角，14px 字重 500，padding `0px 6px`
- 緊湊、藥丸感的分類樣式

## 5. 間距與佈局

### 間距系統
- **基準單位**：8px
- **刻度**：1px、2px、3px、4px、8px、10px、12px、16px、20px、24px、32px、40px
- **區塊 padding**：主要區塊之間垂直 80px–120px
- **卡片 padding**：內部間距 16px–32px
- **元件間距**：相關元素之間 8px–16px

### Grid 與容器
- **最大寬度**：約 1200px 容器（斷點 1204px），置中
- **欄位模式**：單欄 hero、2–3 欄特色 grid、滿版展示區
- **App 展示**：產品 UI 嵌在置中的視窗外框內

### 留白哲學
- **戲劇化的負空間**：區塊漂浮在巨大的深色空間裡，產生具電影感的節奏
- **產品密集、行銷稀疏**：產品 UI 截圖資訊量很高，但周圍的行銷文案使用最少文字搭配大量留白
- **垂直節奏**：區塊內元素之間維持 24px–32px 的間距

### 圓角刻度
- **2px–3px**：微型元素、inline code、極小指示器
- **4px–5px**：鍵盤鍵帽、小型互動元素
- **6px**：按鈕、徽章、tag — 主力圓角
- **8px**：輸入欄位、inline 元件
- **9px–11px**：圖像、中型容器
- **12px**：標準卡片、產品截圖
- **16px**：大型卡片、特色區塊
- **20px**：Hero 卡片、顯眼容器
- **86px+**：藥丸按鈕、導覽 CTA — 完整藥丸形

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|-------|-----------|-----|
| Level 0（虛空） | 無陰影，`#07080a` 表面 | 頁面背景 |
| Level 1（細微） | `rgba(0, 0, 0, 0.28) 0px 1.189px 2.377px` | 最低限度的浮起、inline 元素 |
| Level 2（環形） | 外圈 `rgb(27, 28, 30) 0px 0px 0px 1px` 加內圈 `rgb(7, 8, 10) 0px 0px 0px 1px inset` | 卡片包覆，雙環手法 |
| Level 3（按鈕） | `rgba(255, 255, 255, 0.05) 0px 1px 0px 0px inset` 加 `rgba(255, 255, 255, 0.25) 0px 0px 0px 1px` 加 `rgba(0, 0, 0, 0.2) 0px -1px 0px 0px inset` | macOS 原生按鍵按壓感 — 上緣白色高光、下緣內陰影 |
| Level 4（按鍵） | 五層陰影堆疊加按壓內陰影效果 | 鍵盤快捷鍵鍵帽 — 實體 3D 外觀 |
| Level 5（懸浮） | `rgba(0, 0, 0, 0.5) 0px 0px 0px 2px` 加 `rgba(255, 255, 255, 0.19) 0px 0px 14px` 加內陰影 | 命令選單、懸浮面板 — 厚重深度加上光暈 |

### 陰影哲學
Raycast 的陰影系統是網頁上最接近 macOS 原生質感的。多層陰影組合包含：
- **外環**用於包覆（取代傳統邊框）
- **頂部內陰影高光**（`rgba(255, 255, 255, 0.05–0.25)`）模擬來自上方的光源
- **底部內陰影暗部**（`rgba(0, 0, 0, 0.2)`）模擬下方的陰影
- 效果是實體感的：元素感覺像玻璃或拉絲金屬，而不是平面矩形

### 裝飾性深度
- **暖光**：`rgba(215, 201, 175, 0.05) 0px 0px 20px 5px` 出現在特色元素後方 — 在冷色深色畫布上的細微暖光
- **藍色資訊光暈**：`rgba(0, 153, 255, 0.15)` 用於強調互動狀態
- **紅色危險光暈**：`rgba(255, 99, 99, 0.15)` 用於強調錯誤／破壞性狀態

## 7. Do's and Don'ts

### Do
- 背景使用 `#07080a`（不是純黑） — 那一抹藍色冷調是 Raycast 感的精髓
- 內文加上正字距（+0.2px） — 這是刻意與多數深色 UI 不同的選擇
- 互動元素用多層陰影加內陰影高光 — macOS 原生的深度是招牌
- Raycast Red（`#FF6363`）只當點睛色，不可氾濫 — 留給 hero 與錯誤狀態
- 卡片包覆用 `rgba(255, 255, 255, 0.06)` 邊框 — 幾乎看不見，但結構上不可或缺
- 內文採字重 500 為基準 — 中等字重提升深色模式的可讀性
- 主要 CTA 用藥丸形（86px+ 圓角），次要動作用矩形（6px–8px）
- 所有 Inter 文字啟用 OpenType 的 `calt`、`kern`、`liga`、`ss03`
- 按鈕互動以 opacity 過場（hover 設 opacity 0.6），不要切換顏色

### Don't
- 不要把背景設成純黑（`#000000`） — 藍色冷調才能讓 Raycast 不淪為一般深色主題
- 不要在內文使用負字距 — Raycast 刻意採用正字距以保可讀性
- 不要把 Raycast Blue 當成主要強調色用在所有地方 — 藍色是互動／資訊色，紅色才是品牌色
- 不要做單層平面陰影 — 多層內陰影系統是 macOS 原生美學的核心
- 不要在有 500 可用時用 regular（400）內文 — 多出來的重量能避免深色模式文字顯得單薄
- 不要混用暖色與冷色邊框 — 統一使用冷灰邊框（`hsl(195, 5%, 15%)`）
- 不要只放厚重 drop shadow 而沒有對應的內陰影 — 陰影永遠成對出現（外加內）
- 不要使用裝飾元素、漸層或彩色背景 — 深色虛空是舞台，內容才是表演者

## 8. 響應式行為

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|-------|-------------|
| Mobile | <600px | 單欄、卡片堆疊、漢堡選單、hero 文字縮為約 40px |
| Small Tablet | 600px–768px | 開始出現 2 欄 grid，導覽部分可見 |
| Tablet | 768px–1024px | 2–3 欄特色內容，導覽展開，截圖縮放 |
| Desktop | 1024px–1200px | 完整佈局，所有導覽連結可見，hero 64px |
| Large Desktop | >1200px | 容器最大寬度置中，兩側留有大量邊距 |

### 觸控目標
- 藥丸按鈕：86px 圓角搭配 20px padding — 遠超過 44px 最低要求
- 次要按鈕：最少 8px padding，邊框提供視覺上的點擊範圍延伸
- 導覽連結：16px 文字搭配周圍 padding，確保可觸控

### 收合策略
- **導覽**：水平導覽 → 行動裝置漢堡選單側拉
- **Hero**：64px → 48px → 36px 隨斷點縮放
- **特色 grid**：3 欄 → 2 欄 → 單欄堆疊
- **產品截圖**：在固定比例容器內等比縮放，維持 macOS 視窗外觀的比例
- **鍵盤快捷鍵顯示**：在行動裝置上簡化或隱藏，因為快捷鍵在手機上沒有意義

### 圖像行為
- 產品截圖在固定比例容器內響應式縮放
- Hero 斜紋圖樣等比縮放
- macOS 視窗的圓角在所有尺寸下都維持
- 不顯示懶載入殘影 — 圖像是產品敘事的關鍵

### 迭代指南
精修這套設計系統產出的畫面時：
1. 確認背景是 `#07080a` 而非純黑 — 那一點藍色至關重要
2. 驗證內文採正字距（+0.2px） — 負字距會破壞 Raycast 的美學
3. 確保陰影同時有外層與內層 — 單層陰影看起來會很平板且不對
4. 確認 Inter 已啟用 OpenType 的 `calt`、`kern`、`liga`、`ss03`
5. 測試 hover 狀態使用 opacity 過場（0.6），而非顏色切換 — 這是核心互動模式
