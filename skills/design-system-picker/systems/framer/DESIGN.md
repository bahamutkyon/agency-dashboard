# Framer 啟發的設計系統

> 類別：Design & Creative
> 網站建構工具。大膽的黑藍配色、動效優先、設計感十足。

## 1. 視覺主題與氛圍

Framer 的網站是一塊電影感、極度工具化的深色畫布，散發著「設計師打造的設計工具」的自信。整個體驗浸泡在純黑之中——不是溫暖的炭灰或舒適的深灰，而是絕對的虛無（`#000000`），讓每個元素、每張截圖、每個字型細節都像漂浮在深空之中。這個網站把自家產品 UI 當成主視覺藝術，直接把完整保真度的截圖與互動 demo 嵌入敘事流程。

字型是招牌手法：GT Walsheim 搭配極端緊縮的字距（110px 主視覺甚至用到 -5.5px）造出壓縮、動感、彷彿彈簧蓄勢的標題——像是隨時可能彈開的文字。轉到 Inter 處理內文也毫無接縫，廣泛使用的 OpenType 功能（`cv01`、`cv05`、`cv09`、`cv11`、`ss03`、`ss07`）讓小字也帶有精緻、客製的質感。Framer Blue（`#0099ff`）節制但決斷地運用——作為連結色、邊框點綴、細緻的 ring 陰影——在毫無暖意的黑色中拉出冷峻、電流般的主線。

整體效果是「給網頁設計師的夜店」：黑暗、精確、誘人，毫無歉意地以產品為先。每一個區段都為了展示工具能做什麼而存在，網站本身就是最佳概念驗證。

**核心特徵：**
- 純黑（`#000000`）虛無畫布——絕對黑，不暖、不灰
- GT Walsheim 展示字搭配極端負字距（110px 用 -5.5px）
- Framer Blue（`#0099ff`）作為唯一點綴色——冷峻、電流般、精確
- 藥丸形按鈕（40px–100px 圓角）——互動元素永不出現銳角
- 產品截圖即主視覺藝術——工具本身就是行銷
- 霧面玻璃按鈕變體在深色表面上使用 `rgba(255, 255, 255, 0.1)`
- 廣泛在 Inter 上使用 OpenType 功能做精緻的微字型

## 2. 配色系統與角色

### 主要色
- **Pure Black** (`#000000`)：主背景，定義 Framer 深色優先識別的虛無畫布
- **Pure White** (`#ffffff`)：深色表面上的主要文字色、點綴背景上的按鈕文字
- **Framer Blue** (`#0099ff`)：主要點綴色——連結、邊框、ring 陰影、互動高光

### 次要與點綴
- **Muted Silver** (`#a6a6a6`)：次要文字、低調標籤、深色表面上的弱化說明
- **Near Black** (`#090909`)：升起的深色表面、用於細緻深度分隔的 ring 陰影色

### 表面與背景
- **Void Black** (`#000000`)：頁面背景、主畫布
- **Frosted White** (`rgba(255, 255, 255, 0.1)`)：半透明按鈕背景、深色上的玻璃表面
- **Subtle White** (`rgba(255, 255, 255, 0.5)`)：更不透明的霧面元素，用於 hover 狀態

### 中性與文字
- **Pure White** (`#ffffff`)：標題文字、高強調內文
- **Muted Silver** (`#a6a6a6`)：內文、說明、次要資訊
- **Ghost White** (`rgba(255, 255, 255, 0.6)`)：第三層文字、深色表面上的 placeholder

### 語意與點綴
- **Framer Blue** (`#0099ff`)：連結、互動邊框、focus ring
- **Blue Glow** (`rgba(0, 153, 255, 0.15)`)：focus ring 陰影、互動元素周圍細緻的藍色光暈
- **Default Link Blue** (`#0000ee`)：標準瀏覽器連結色（在內容區節制使用）

### 漸層系統
- 無顯著漸層使用——Framer 依賴純黑表面，偶爾用藍色帶光暈做深度
- 產品截圖背後使用 Framer Blue 極低不透明度的細緻徑向光暈

## 3. 字型系統

### 字型家族
- **展示字型**：`GT Walsheim Framer Medium` / `GT Walsheim Medium`——自訂幾何無襯線、字重 500。備援：`GT Walsheim Framer Medium Placeholder`、系統無襯線
- **內文／UI**：`Inter Variable` / `Inter`——可變無襯線，廣泛使用 OpenType 功能。備援：`Inter Placeholder`、`-apple-system`、`system-ui`
- **點綴字型**：`Mona Sans`——GitHub 開源字型，在特定元素以超細字重（100）使用
- **等寬字型**：`Azeret Mono`——程式碼與技術標籤的等寬夥伴
- **圓潤字型**：`Open Runde`——小型圓潤的微標籤夥伴字

### 原則
- **壓縮即個性**：GT Walsheim 的極端負字距（110px 用 -5.5px）是定義性的字型手法——標題感覺像彈簧蓄勢、急迫、近乎屏息
- **OpenType 極致主義**：Inter 同時啟用 6+ 個 OpenType 功能（`cv01`、`cv05`、`cv09`、`cv11`、`ss03`、`ss07`），即使在內文尺寸也透出細緻的客製感
- **展示字字重節制**：所有 GT Walsheim 使用 500 字重（medium）——絕不粗體、絕不 regular。建立自信但不咄咄逼人的展示語氣
- **超緊行高**：展示文字行高 0.85 意味字母在垂直方向幾乎重疊——刻意的密度，獎勵在合適距離閱讀

## 4. 元件樣式

### 按鈕
- **霧面藥丸**：`rgba(255, 255, 255, 0.1)` 背景、黑色文字（`#000000`）、藥丸形（40px 圓角）。在深色表面上活的玻璃效果按鈕——半透明、環境、低調
- **實心白藥丸**：`rgb(255, 255, 255)` 背景、黑色文字（`#000000`）、完整藥丸（100px 圓角）、padding `10px 15px`。主要 CTA——乾淨、深色上高對比、不容錯過
- **Ghost**：無可見背景、白色文字，純靠文字樣式。Hover 時顯露細緻的霧面背景
- **過渡**：基於縮放的動畫（matrix 變換、0.85 縮放比例）、不透明度過渡用於顯露效果

### 卡片與容器
- **深色表面卡片**：黑色或近黑（`#090909`）背景、`rgba(0, 153, 255, 0.15) 0px 0px 0px 1px` 藍色 ring 陰影邊框、圓角（10px–15px）
- **升起卡片**：多層陰影——`rgba(255, 255, 255, 0.1) 0px 0.5px 0px 0.5px`（細緻頂緣高光）＋ `rgba(0, 0, 0, 0.25) 0px 10px 30px`（深層環境陰影）
- **產品截圖**：在深色容器內全寬或加 padding、8px–12px 圓角用於軟體 UI 預覽
- **Hover**：Framer Blue ring 陰影輕微增強，或在霧面表面提升亮度

### 輸入與表單
- 行銷網站表單很少出現
- 輸入欄位遵循深色主題：深色背景、細邊框、白色文字
- Focus 狀態：Framer Blue（`#0099ff`）ring 邊框、`1px solid #0099ff`
- Placeholder 文字 `rgba(255, 255, 255, 0.4)`

### 導覽
- **深色浮動導覽列**：黑色背景帶霧面玻璃效果、白色文字連結
- **導覽連結**：Inter 15px 字重 400、白色文字、hover 細微不透明度變化
- **CTA 按鈕**：藥丸形、白色或霧面、位於導覽右端
- **手機**：摺疊為漢堡選單，維持深色主題
- **黏性行為**：捲動時導覽固定於頂端

### 圖像處理
- **產品截圖即主視覺**：全寬嵌入的 UI 截圖搭配圓角（8px–12px）
- **深色對深色構圖**：截圖放在黑色背景上，用細緻陰影做深度分隔
- **16:9 與自訂長寬比**：產品 demo 填滿容器
- **無裝飾性圖像**：所有圖像都有功能——展示工具、輸出、工作流程

### 信任與社會證明
- 客戶 logo 與評語以深色表面上的弱化灰呈現
- 最少裝飾——產品截圖就是信任訊號

## 5. 間距與佈局

### 間距系統
- **基本單位**：8px
- **尺度**：1px、2px、3px、4px、5px、6px、8px、10px、12px、15px、20px、30px、35px
- **區段 padding**：區段之間大幅垂直間距（80px–120px）
- **卡片內距**：15px–30px 內部 padding
- **元件間距**：相關元素之間 8px–20px

### 網格與容器
- **最大寬度**：約 1200px 容器、置中
- **欄位模式**：全寬 hero、2 欄功能區段、單欄產品展示
- **不對稱佈局**：功能區段常配 40% 文字 + 60% 截圖

### 留白哲學
- **以黑暗呼吸**：區段之間寬裕的垂直間距——黑色背景使留白化為虛無，在內容區塊之間製造戲劇性的停頓
- **內密、外鬆**：個別元件構圖緊湊（緊行高、壓縮文字），卻漂浮在寬裕的周圍空間中
- **產品優先的密度**：截圖區可以密集、資訊豐富，與稀疏的行銷文字形成對比

### 圓角尺度
- **1px**：微元素、近乎方正的精準邊
- **5px–7px**：小型 UI 元素、圖片縮圖——略微柔化
- **8px**：標準元件圓角——程式碼區塊、按鈕、互動元素
- **10px–12px**：卡片、產品截圖——舒適圓潤
- **15px–20px**：大型容器、功能卡片——寬裕圓潤
- **30px–40px**：導覽藥丸、分頁——明顯圓潤
- **100px**：完整藥丸形——主要 CTA、標籤元素

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|------|----------|------|
| Level 0（平面） | 無陰影、純黑表面 | 頁面背景、空白區域 |
| Level 1（Ring） | `rgba(0, 153, 255, 0.15) 0px 0px 0px 1px` | 卡片邊框、互動元素外框——Framer Blue 光暈 ring |
| Level 2（收束） | `rgb(9, 9, 9) 0px 0px 0px 2px` | 深色表面上細緻收束的近黑 ring |
| Level 3（浮動） | `rgba(255, 255, 255, 0.1) 0px 0.5px 0px 0.5px, rgba(0, 0, 0, 0.25) 0px 10px 30px` | 升起卡片、浮動元素——細緻頂緣白色高光 + 深層環境陰影 |

### 陰影哲學
Framer 的層次系統與傳統淺色主題設計相反。Framer 不在淺色背景上用較深陰影，而是：
- **藍色 ring 陰影**極低不透明度（0.15）用於收束——一個招牌手法，細緻地為每個帶邊框元素打上品牌印記
- **白色邊緣高光**（0.5px）在升起元素的頂端——模擬光打在頂面上
- **深層環境陰影**用於真正浮動的元素——`rgba(0, 0, 0, 0.25)` 大範圍擴散（30px）

### 裝飾性深度
- **藍色光暈**：關鍵互動區後方細緻的 Framer Blue（`#0099ff`）徑向漸層
- **無背景模糊／玻璃擬態**：儘管按鈕有霧面效果，並未使用厚重的玻璃模糊——半透明只靠簡單的 rgba 不透明度

## 7. Do's and Don'ts

### Do
- 用純黑（`#000000`）作為主要背景——不是深灰、不是炭灰
- 在 GT Walsheim 展示文字上套用極端負字距（-3px 到 -5.5px）
- 所有按鈕保持藥丸形（40px+ 圓角）——絕不用方形或微圓的按鈕
- Framer Blue（`#0099ff`）只用於互動點綴——連結、邊框、focus 狀態
- 深色背景上用 `rgba(255, 255, 255, 0.1)` 做霧面玻璃表面
- GT Walsheim 只用 500 字重——medium 字重就是品牌
- 在 Inter 文字上廣泛使用 OpenType 功能（cv01、cv05、cv09、cv11、ss03、ss07）
- 讓產品截圖成為視覺中心——工具自己行銷自己
- 用藍色 ring 陰影（`rgba(0, 153, 255, 0.15) 0px 0px 0px 1px`）做卡片收束

### Don't
- 不要用暖色深色背景（不要 `#1a1a1a`、`#2d2d2d` 或棕黑）
- 不要對 GT Walsheim 展示文字套用粗體（700+）——只用 medium 500
- 不要在 Framer Blue 之外引入其他點綴色——這是單一點綴色系統
- 不要對非互動元素用大圓角（卡片用 10px–15px，只有按鈕能 40px+）
- 不要加裝飾性圖像、插畫或圖示——產品就是插畫
- 不要對標題用正字距——一切都是壓縮的負字距
- 不要做厚重的 drop shadow——深度透過細緻 ring 與微量環境陰影傳達
- 不要在內容區段後面放淺色／白色背景——虛無是神聖的
- 不要用襯線或展示字重的字型——系統只用幾何無襯線

## 8. 響應式行為

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|------|----------|
| Mobile | <809px | 單欄、功能區段堆疊、hero 文字縮減（62px→40px）、漢堡導覽 |
| Tablet | 809px–1199px | 開始出現 2 欄功能、導覽連結部分可見、截圖縮小 |
| Desktop | >1199px | 完整佈局、展開導覽含所有連結＋CTA、110px 展示 hero、並排功能 |

### 觸控目標
- 藥丸按鈕：最小 40px 高、10px 垂直 padding——超過 WCAG 最小 44px 標準
- 導覽連結：15px 文字搭配寬裕 padding 以利觸控
- 手機 CTA 按鈕：手機上全寬藥丸，方便拇指觸及

### 摺疊策略
- **導覽**：完整水平導覽 → 手機斷點變漢堡選單
- **Hero 文字**：110px 展示 → 85px → 62px → 約 40px 跨斷點縮減，等比例維持極端負字距
- **功能區段**：並排（文字＋截圖）→ 手機垂直堆疊
- **產品截圖**：在容器邊界內響應式縮放，維持長寬比
- **區段間距**：等比縮減——桌面 120px → 手機 60px

### 圖像行為
- 產品截圖響應式，在容器邊界內縮放
- 無藝術指導變更——所有斷點同樣的裁切
- 深色背景確保截圖在任何尺寸都維持視覺衝擊
- 截圖在使用者捲入視野時 lazy-load

## 9. Agent Prompt 指南

### 快速色彩參考
- 主要背景：Void Black（`#000000`）
- 主要文字：Pure White（`#ffffff`）
- 點綴／CTA：Framer Blue（`#0099ff`）
- 次要文字：Muted Silver（`#a6a6a6`）
- 霧面表面：半透明白（`rgba(255, 255, 255, 0.1)`）
- 層次 Ring：藍色光暈（`rgba(0, 153, 255, 0.15)`）

### 範例元件 Prompt
- 「在純黑背景上建立 hero 區段，110px GT Walsheim 標題用白色、字距 -5.5px、行高 0.85，搭配白色藥丸 CTA 按鈕（100px 圓角）和黑色文字」
- 「設計黑色背景上的功能卡片，1px Framer Blue ring 陰影邊框（`rgba(0,153,255,0.15)`）、12px 圓角、白色標題用 Inter 22px 字重 700、淺銀（`#a6a6a6`）內文」
- 「打造黑色背景的導覽列，白色 Inter 連結 15px、霧面藥丸按鈕（`rgba(255,255,255,0.1)` 背景、40px 圓角）作為 CTA」
- 「建立產品展示區段，全寬截圖嵌入黑色上、10px 圓角、細緻多層陰影（白色 0.5px 頂緣高光 + `rgba(0,0,0,0.25)` 30px 環境）」
- 「設計定價卡：純黑表面、Framer Blue（`#0099ff`）點綴用於選中方案邊框、白色字型階層（24px Inter 粗體標題、14px regular 內文）、實心白色藥丸 CTA 按鈕」

### 迭代指南
精修此設計系統生成的畫面時：
1. 一次專注一個元件——深色畫布讓每個元素都珍貴
2. 永遠檢查 GT Walsheim 標題的字距——極端負字距不可妥協
3. 確認 Framer Blue 只出現在互動元素上——絕不作為裝飾背景或非連結文字色
4. 確保所有按鈕都是藥丸形——任何方角立刻打破 Framer 美學
5. 測試霧面玻璃表面是否確切為 `rgba(255, 255, 255, 0.1)`——太不透明會像 bug、太透明會消失
