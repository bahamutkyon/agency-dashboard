# Sentry 啟發的設計系統

> 類別：Backend & Data
> 錯誤監控。深色儀表板、資料密集、粉紫色點綴。

## 1. 視覺主題與氛圍

Sentry 的網站是一個深色模式優先的開發者工具介面，講的是程式碼編輯器與終端機視窗的語言。整體美學紮根於深邃的紫黑背景（`#1f1633`、`#150f23`），喚起 Sentry 為之而生的深夜除錯時光。在這片墨黑畫布上，精心挑選的紫、粉與獨特的萊姆綠點綴（`#c2ef4e`）構成一個既技術又鮮活的視覺系統。

字型搭配是刻意的：「Dammit Sans」以主視覺尺寸登場（88px、字重 700），作為帶有個性與態度的展示字型，呼應 Sentry 不正經的品牌語氣（「Code breaks. Fix it faster.」）；Rubik 則擔任所有功能性文字的主力 UI 字型——標題、內文、按鈕、說明、導覽都是它。Monaco 提供等寬層級給程式碼片段與技術內容，完成開發者工具的字型三位一體。

Sentry 的獨到之處在於它擁抱「深色 IDE」美學卻不顯冰冷無情。溫暖的紫色取代了開發者工具典型的冷灰色，大膽的插畫元素（3D 角色、繽紛的產品截圖）為深色畫布增添節奏。按鈕系統使用招牌的悶紫色（`#79628c`）搭配內陰影，創造出觸感、近乎實體的質地——按鈕彷彿真的能被按進表面裡。

**核心特徵：**
- 紫黑色背景（`#1f1633`、`#150f23`）——絕不純黑
- 暖色紫色光譜：從深（`#362d59`）經中（`#79628c`、`#6a5fc1`）到鮮明（`#422082`）
- 萊姆綠點綴（`#c2ef4e`）用於高可見度 CTA 與重點
- 粉／珊瑚色點綴（`#ffb287`、`#fa7faa`）用於 focus 狀態與次要重點
- 「Dammit Sans」展示字型在主視覺尺寸傳達品牌個性
- Rubik 為主要 UI 字型，搭配大寫字距處理的標籤
- Monaco 等寬字用於程式碼元素
- 按鈕的內陰影創造觸感深度
- 採用 `blur(18px) saturate(180%)` 的霧面玻璃效果

## 2. 配色系統與角色

### 主要品牌色
- **Deep Purple** (`#1f1633`)：主背景，定義品牌的色彩
- **Darker Purple** (`#150f23`)：更深的區段、頁腳、次要背景
- **Border Purple** (`#362d59`)：邊框、分隔線、細緻的結構線

### 點綴色
- **Sentry Purple** (`#6a5fc1`)：主互動色——連結、hover、focus ring
- **Muted Purple** (`#79628c`)：按鈕背景、次要互動元素
- **Deep Violet** (`#422082`)：下拉選單、啟用狀態、高強調表面
- **Lime Green** (`#c2ef4e`)：高可見度點綴、特殊連結、徽章高光
- **Coral** (`#ffb287`)：focus 狀態背景、暖色點綴
- **Pink** (`#fa7faa`)：focus 外框、裝飾點綴

### 文字色
- **Pure White** (`#ffffff`)：深色背景上的主要文字
- **Light Gray** (`#e5e7eb`)：次要文字、弱化內容
- **Code Yellow** (`#dcdcaa`)：語法高亮、程式碼 token

### 表面與覆蓋層
- **Glass White** (`rgba(255, 255, 255, 0.18)`)：霧面玻璃按鈕背景
- **Glass Dark** (`rgba(54, 22, 107, 0.14)`)：玻璃元素的 hover 覆蓋層
- **Input White** (`#ffffff`)：表單輸入背景（淺色情境）
- **Input Border** (`#cfcfdb`)：表單欄位邊框

### 陰影
- **Ambient Glow** (`rgba(22, 15, 36, 0.9) 0px 4px 4px 9px`)：深紫色環境陰影
- **Button Hover** (`rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem`)：升起的 hover 狀態
- **Card Shadow** (`rgba(0, 0, 0, 0.1) 0px 10px 15px -3px`)：標準卡片層次
- **Inset Button** (`rgba(0, 0, 0, 0.1) 0px 1px 3px 0px inset`)：觸感按壓效果

## 3. 字型系統

### 字型家族
- **展示字型**：`Dammit Sans`——主視覺標題用的品牌個性字
- **主要 UI 字型**：`Rubik`，備援：`-apple-system, system-ui, Segoe UI, Helvetica, Arial`
- **等寬字型**：`Monaco`，備援：`Menlo, Ubuntu Mono`

### 階層

| 角色 | 字型 | 字級 | 字重 | 行高 | 字距 | 備註 |
|------|------|------|------|------|------|------|

| Section Heading | Rubik | 30px (1.88rem) | 400 | 1.20（緊） | normal | 主要區段標題 |
| Sub-heading | Rubik | 27px (1.69rem) | 500 | 1.25（緊） | normal | 功能區段標題 |
| Card Title | Rubik | 24px (1.50rem) | 500 | 1.25（緊） | normal | 卡片與區塊標題 |

### 原則
- **雙重個性**：Dammit Sans 在展示尺寸帶來不正經的品牌性格；Rubik 為所有功能元素提供乾淨的專業感。
- **大寫作為系統**：按鈕、說明、標籤與微型文字全部使用 `text-transform: uppercase` 並帶細緻字距（0.2px–0.25px），形成貫穿全站的「技術標籤」模式。
- **字重分層**：Rubik 使用 400（內文）、500（強調／導覽）、600（標題／強調）、700（按鈕／CTA）——清晰的四階字重系統。
- **緊湊標題、放鬆內文**：所有標題使用 1.10–1.25 行高；內文用 1.50；小型說明在極小字級下放寬到 2.00 以確保可讀性。

## 4. 元件樣式

### 按鈕

**主要悶紫**
- 背景：`#79628c` (rgb(121, 98, 140))
- 文字：`#ffffff`，大寫、14px、字重 500–700、字距 0.2px
- 邊框：`1px solid #584674`
- 圓角：13px
- 陰影：`rgba(0, 0, 0, 0.1) 0px 1px 3px 0px inset`（觸感內陰影）
- Hover：升起的陰影 `rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem`

**玻璃白**
- 背景：`rgba(255, 255, 255, 0.18)`（霧面玻璃）
- 文字：`#ffffff`
- Padding：8px
- 圓角：12px（左對齊變體：`12px 0px 0px 12px`）
- 陰影：`rgba(0, 0, 0, 0.08) 0px 2px 8px`
- Hover 背景：`rgba(54, 22, 107, 0.14)`
- 用途：深色表面上的次要動作

**純白實心**
- 背景：`#ffffff`
- 文字：`#1f1633`
- Padding：12px 16px
- 圓角：8px
- Hover：背景過渡至 `#6a5fc1`、文字轉白
- Focus：背景 `#ffb287`（珊瑚色）、外框 `rgb(106, 95, 193) solid 0.125rem`
- 用途：深色背景上的高可見度 CTA

**深紫羅蘭（下拉選單）**
- 背景：`#422082`
- 文字：`#ffffff`
- Padding：8px 16px
- 圓角：8px

### 輸入框

**文字輸入**
- 背景：`#ffffff`
- 文字：`#1f1633`
- 邊框：`1px solid #cfcfdb`
- Padding：8px 12px
- 圓角：6px
- Focus：邊框色維持 `#cfcfdb`、陰影 `rgba(0, 0, 0, 0.15) 0px 2px 10px inset`

### 連結
- **深色上預設**：`#ffffff`，底線裝飾
- **Hover**：色彩過渡至 `#6a5fc1`（Sentry Purple）
- **紫色連結**：預設 `#6a5fc1`、hover 加底線
- **萊姆綠點綴連結**：預設 `#c2ef4e`、hover 至 `#6a5fc1`
- **深色情境連結**：`#362d59`、hover 至 `#ffffff`

### 卡片與容器
- 背景：半透明或深紫表面
- 圓角：8px–12px
- 陰影：`rgba(0, 0, 0, 0.1) 0px 10px 15px -3px`
- 背景濾鏡：玻璃效果用 `blur(18px) saturate(180%)`

### 導覽
- 在 hero 內容上方的深色透明 header
- Rubik 15px 字重 500 用於導覽連結
- 白色文字、hover 至 Sentry Purple（`#6a5fc1`）
- 類別使用大寫並帶 0.2px 字距
- 手機版：漢堡選單、全寬展開

## 5. 間距與佈局

### 間距系統
- 基本單位：8px
- 尺度：1px、2px、4px、5px、6px、8px、12px、16px、24px、32px、40px、44px、45px、47px

### 網格與容器
- 最大內容寬度：1152px（XL 斷點）
- 響應式 padding：2rem（手機）→ 4rem（平板以上）
- 容器內內容置中
- 全寬深色區段搭配收束的內部內容

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|------|----------|
| Mobile | < 576px | 單欄、堆疊佈局 |
| Small Tablet | 576–640px | 微調寬度 |
| Tablet | 640–768px | 開始出現 2 欄 |
| Small Desktop | 768–992px | 完整導覽可見 |
| Desktop | 992–1152px | 標準佈局 |
| Large Desktop | 1152–1440px | 最大寬度內容 |

### 留白哲學
- **深色呼吸空間**：區段之間有寬裕的垂直間距（64px–80px+），讓深色背景成為視覺的休息點。
- **內容孤島**：功能區段是漂浮在深紫汪洋上的自足區塊，各自擁有內部間距節奏。
- **不對稱的 padding**：按鈕採用不對稱 padding（12px 16px、8px 12px），感覺有機而非僵硬。

### 圓角尺度
- 最小（6px）：表單輸入、小型互動元素
- 標準（8px）：按鈕、卡片、容器
- 舒適（10px–12px）：較大的容器、玻璃面板
- 圓潤（13px）：主要悶紫按鈕
- 藥丸（18px）：圖片容器、徽章

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|------|----------|------|
| 沉陷（Level -1） | 內陰影 `rgba(0, 0, 0, 0.1) 0px 1px 3px inset` | 主要按鈕（觸感按壓） |
| 平面（Level 0） | 無陰影 | 預設表面、深色背景 |
| 表面（Level 1） | `rgba(0, 0, 0, 0.08) 0px 2px 8px` | 玻璃按鈕、細膩卡片 |
| 升起（Level 2） | `rgba(0, 0, 0, 0.1) 0px 10px 15px -3px` | 卡片、浮動面板 |
| 突出（Level 3） | `rgba(0, 0, 0, 0.18) 0px 0.5rem 1.5rem` | Hover 狀態、彈窗 |
| 環境（Level 4） | `rgba(22, 15, 36, 0.9) 0px 4px 4px 9px` | Hero 周圍深紫環境暈光 |

**陰影哲學**：Sentry 採用獨特的組合——內陰影（按鈕彷彿按進表面）與環境暈光（內容從深色背景中散發）。深紫色環境陰影（`rgba(22, 15, 36, 0.9)`）是招牌——它形成生物發光般的質感，讓內容像是自體發出紫色調的光。

## 7. Do's and Don'ts

### Do
- 使用深紫背景（`#1f1633`、`#150f23`）——絕不純黑（`#000000`）
- 在主要按鈕上加內陰影以製造觸感按壓效果
- Dammit Sans 只用於主視覺／展示標題——其他一律 Rubik
- 在按鈕與標籤上套用 `text-transform: uppercase` 並加 `letter-spacing: 0.2px`
- 萊姆綠點綴（`#c2ef4e`）節制使用以製造最大衝擊
- 使用霧面玻璃效果（`blur(18px) saturate(180%)`）做層疊表面
- 維持暖紫色的陰影色調——陰影應帶紫調而非中性灰
- 使用 Rubik 的四階字重系統：400（內文）、500（導覽／強調）、600（標題）、700（CTA）

### Don't
- 不要用純黑（`#000000`）當背景——永遠用暖紫黑
- 不要把 Dammit Sans 用在內文或 UI 元素——它是 display-only
- 不要用標準灰（`#666`、`#999`）做邊框——用帶紫調的灰（`#362d59`、`#584674`）
- 不要在按鈕拿掉大寫處理——這是全站系統模式
- 不要用銳角（0px 圓角）——所有互動元素至少 6px
- 不要在同一個元件混用萊姆綠與珊瑚／粉色點綴
- 不要在主要按鈕用扁平（非內陰影）陰影——觸感是招牌
- 大寫文字別忘了字距——至少 0.2px

- 容器 padding：4rem → 2rem

## 9. Agent Prompt 指南

### 快速色彩參考
- 背景：`#1f1633`（主要）、`#150f23`（更深）
- 文字：`#ffffff`（主要）、`#e5e7eb`（次要）
- 互動：`#6a5fc1`（連結／hover）、`#79628c`（按鈕）
- 點綴：`#c2ef4e`（萊姆高光）、`#ffb287`（珊瑚 focus）
- 邊框：`#362d59`（深色）、`#cfcfdb`（淺色情境）

### 範例元件 Prompt
- 「在深紫背景（`#1f1633`）上建立 hero 區段。標題用 88px Dammit Sans 字重 700、行高 1.20、白色文字。副標 16px Rubik 字重 400、行高 1.50。白色實心 CTA 按鈕（8px 圓角、12px 16px padding），hover 過渡至 `#6a5fc1`。」
- 「設計導覽列：在深色背景上透明。Rubik 15px 字重 500、白色文字。大寫類別標籤帶 0.2px 字距。Hover 色 `#6a5fc1`。」
- 「打造主要按鈕：背景 `#79628c`、邊框 `1px solid #584674`、內陰影 `rgba(0,0,0,0.1) 0px 1px 3px`、白色大寫文字 14px Rubik 字重 700、字距 0.2px、圓角 13px。Hover：陰影 `rgba(0,0,0,0.18) 0px 0.5rem 1.5rem`。」
- 「建立玻璃卡片面板：背景 `rgba(255,255,255,0.18)`、backdrop-filter `blur(18px) saturate(180%)`、圓角 12px。內部白色文字內容。」
- 「設計功能區段：`#150f23` 背景、24px Rubik 字重 500 標題、16px Rubik 字重 400 內文。標題上方 14px 大寫萊姆綠（`#c2ef4e`）標籤。」

### 迭代指南
1. 永遠從深紫背景開始——色盤是為深色模式打造
2. 按鈕用內陰影、hero 區段用環境紫光
3. 大寫＋字距是標籤、按鈕、說明的系統模式
4. 萊姆綠（`#c2ef4e`）是「亮點」色——每個區段最多用一次
5. 疊層面板用霧面玻璃、主要表面用實心紫色
6. Rubik 處理 90% 的字型——Dammit Sans 只給 hero
