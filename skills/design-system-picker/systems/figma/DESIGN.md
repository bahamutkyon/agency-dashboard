# Figma 啟發的設計系統

> 類別：Design & Creative
> 協作設計工具。鮮豔多彩、俏皮卻專業。

## 1. 視覺主題與氛圍

Figma 的介面是「自己設計自己」的設計工具——字型運用的精緻教科書。一款自訂的可變字型（figmaSans）在極細（字重 320）與粗體（字重 700）之間調節，並停在多數字型系統從不探索的特殊中間值（330、340、450、480、540）。這種顆粒化的字重控制讓每個文字元素都有精準校準的視覺重量，透過微差而非「regular vs bold」這種粗糙工具建立階層。

頁面呈現迷人的二元性：介面框架嚴格只用黑白（實際偵測到的顏色只有 `#000000` 與 `#ffffff`），而 hero 區段與產品展示則用鮮豔的多色漸層爆發——電光綠、明黃、深紫、桃紅。這種分離意味著設計系統本身是無色的，把產品多彩的輸出當成主視覺內容。Figma 的行銷頁本質上就是一面白色畫廊牆，展示繽紛的藝術。

讓 Figma 與眾不同的不只是可變字型，還有它的圓形與藥丸幾何。按鈕用 50px 圓角（藥丸）或 50%（完美圓形給圖示按鈕），創造出有機、像工具調色盤的感覺。虛線外框的 focus 指示器（`dashed 2px`）是刻意的設計選擇，呼應 Figma 編輯器中的選取手把——網站的 UI 語言反過來指涉產品的 UI 語言。

**核心特徵：**
- 自訂可變字型（figmaSans）搭配不尋常的字重停點：320、330、340、450、480、540、700
- 介面框架嚴格黑白——色彩只出現在產品內容
- figmaMono 用於大寬字距的大寫技術標籤
- 藥丸（50px）與圓形（50%）按鈕幾何
- 虛線 focus 外框呼應 Figma 編輯器的選取手把
- 鮮豔的多色 hero 漸層（綠、黃、紫、粉）
- 全域啟用 OpenType `"kern"` 功能
- 全站負字距——連內文都用 -0.14px 到 -0.26px

## 2. 配色系統與角色

### 主要色
- **Pure Black** (`#000000`)：所有文字、所有實心按鈕、所有邊框。介面的唯一「顏色」。
- **Pure White** (`#ffffff`)：所有背景、白色按鈕、深色表面上的文字。二元的另一半。

*註：Figma 行銷網站的介面層只用這兩個顏色。所有鮮豔色彩只出現在產品截圖、hero 漸層與嵌入內容中。*

### 表面與背景
- **Pure White** (`#ffffff`)：主頁面背景與卡片表面。
- **Glass Black** (`rgba(0, 0, 0, 0.08)`)：次要圓形按鈕與玻璃效果的細緻深色覆蓋層。
- **Glass White** (`rgba(255, 255, 255, 0.16)`)：深色／彩色表面上按鈕的霧面玻璃覆蓋層。

### 漸層系統
- **Hero 漸層**：使用電光綠、明黃、深紫、桃紅的鮮豔多停漸層。這個漸層是 hero 區段的視覺簽名——代表工具的創意可能性。
- **產品區段漸層**：個別產品區（Design、Dev Mode、Prototyping）在展示中可能使用各自的色彩主題。

## 3. 字型系統

### 字型家族
- **主要**：`figmaSans`，備援：`figmaSans Fallback, SF Pro Display, system-ui, helvetica`
- **等寬／標籤**：`figmaMono`，備援：`figmaMono Fallback, SF Mono, menlo`

### 階層

| 角色 | 字型 | 字級 | 字重 | 行高 | 字距 | 備註 |
|------|------|------|------|------|------|------|
| Display / Hero | figmaSans | 86px (5.38rem) | 400 | 1.00（緊） | -1.72px | 最大衝擊、極端字距 |
| Section Heading | figmaSans | 64px (4rem) | 400 | 1.10（緊） | -0.96px | 功能區段標題 |
| Sub-heading | figmaSans | 26px (1.63rem) | 540 | 1.35 | -0.26px | 強調的區段文字 |
| Sub-heading Light | figmaSans | 26px (1.63rem) | 340 | 1.35 | -0.26px | 輕字重的區段文字 |
| Feature Title | figmaSans | 24px (1.5rem) | 700 | 1.45 | normal | 粗體卡片標題 |
| Body Large | figmaSans | 20px (1.25rem) | 330–450 | 1.30–1.40 | -0.1px 到 -0.14px | 說明、引言 |
| Body / Button | figmaSans | 16px (1rem) | 330–400 | 1.40–1.45 | -0.14px 到 normal | 標準內文、導覽、按鈕 |
| Body Light | figmaSans | 18px (1.13rem) | 320 | 1.45 | -0.26px 到 normal | 輕字重內文 |
| Mono Label | figmaMono | 18px (1.13rem) | 400 | 1.30（緊） | 0.54px | 大寫區段標籤 |
| Mono Small | figmaMono | 12px (0.75rem) | 400 | 1.00（緊） | 0.6px | 大寫小標籤 |

### 原則
- **可變字型的精準**：figmaSans 使用多數系統從不觸及的字重——320、330、340、450、480、540。透過細膩的字重差異而非戲劇化跳階建立階層。330 與 340 的差異幾乎難以察覺，但結構上意義重大。
- **以輕為基底**：多數內文使用 320–340（比典型的 400「regular」更輕），創造空靈、通透的閱讀體驗，呼應設計工具的美學。
- **到處啟用 Kern**：每個文字元素都啟用 OpenType `"kern"` 功能——字距不是選項，而是結構性的。
- **預設負字距**：連內文都使用 -0.1px 到 -0.26px 字距，創造全站緊湊的文字。展示文字進一步壓縮到 -0.96px 與 -1.72px。
- **等寬作為結構**：figmaMono 大寫並帶正字距（0.54px–0.6px）形成技術指路標籤。

## 4. 元件樣式

### 按鈕

**黑色實心（藥丸）**
- 背景：純黑（`#000000`）
- 文字：純白（`#ffffff`）
- 圓角：圓形（50%）給圖示按鈕
- Focus：dashed 2px 外框
- 最大強調

**白色藥丸**
- 背景：純白（`#ffffff`）
- 文字：純黑（`#000000`）
- Padding：8px 18px 10px（不對稱垂直）
- 圓角：藥丸（50px）
- Focus：dashed 2px 外框
- 深色／彩色表面上的標準 CTA

**玻璃深色**
- 背景：`rgba(0, 0, 0, 0.08)`（細緻深色覆蓋層）
- 文字：純黑
- 圓角：圓形（50%）
- Focus：dashed 2px 外框
- 淺色表面上的次要動作

**玻璃淺色**
- 背景：`rgba(255, 255, 255, 0.16)`（霧面玻璃）
- 文字：純白
- 圓角：圓形（50%）
- Focus：dashed 2px 外框
- 深色／彩色表面上的次要動作

### 卡片與容器
- 背景：純白
- 邊框：無或極少
- 圓角：6px（小容器）、8px（圖片、卡片、對話框）
- 陰影：細緻到中等的層次效果
- 產品截圖作為卡片內容

### 導覽
- 白色上的乾淨水平導覽
- Logo：黑色 Figma 字標
- 產品頁籤：藥丸形（50px）的頁籤導覽
- 連結：黑色文字、1px 底線裝飾
- CTA：黑色藥丸按鈕
- Hover：透過 CSS 變數做文字色變化

### 獨特元件

**產品頁籤列**
- 水平藥丸形頁籤（50px 圓角）
- 每個頁籤代表 Figma 的一個產品區（Design、Dev Mode、Prototyping 等）
- 啟用頁籤被高亮

**Hero 漸層區段**
- 全寬鮮豔多色漸層背景
- 白色文字覆蓋搭配 86px 展示標題
- 產品截圖漂浮於漸層中

**虛線 Focus 指示器**
- 所有互動元素 focus 時使用 `dashed 2px` 外框
- 呼應 Figma 編輯器的選取手把
- 連結網站與產品的後設設計選擇

## 5. 間距與佈局

### 間距系統
- 基本單位：8px
- 尺度：1px、2px、4px、4.5px、8px、10px、12px、16px、18px、24px、32px、40px、46px、48px、50px

### 網格與容器
- 最大容器寬度：最寬 1920px
- Hero：全寬漸層搭配置中內容
- 產品區段：交替的展示
- 頁腳：深色全寬區段
- 響應式範圍 559px 到 1920px

### 留白哲學
- **畫廊般的節奏**：寬裕的間距讓每個產品區段像獨立展品般呼吸。
- **彩色區段作為視覺呼吸**：漸層 hero 與產品展示在單色介面區段間提供色彩緩衝。

### 圓角尺度
- 最小（2px）：小型連結元素
- 細緻（6px）：小容器、分隔
- 舒適（8px）：卡片、圖片、對話框
- 藥丸（50px）：頁籤按鈕、CTA
- 圓形（50%）：圖示按鈕、圓形元素

## 6. 深度與層次

| 層級 | 處理方式 | 用途 |
|------|----------|------|
| 平面（Level 0） | 無陰影 | 頁面背景、大多數文字 |
| 表面（Level 1） | 漸層／深色區段上的白色卡片 | 卡片、產品展示 |
| 升起（Level 2） | 細緻陰影 | 浮動卡片、hover 狀態 |

**陰影哲學**：Figma 節制使用陰影。主要的深度機制是**背景對比**（彩色／深色區段上的白色內容）與產品截圖本身的固有立體感。

## 7. Do's and Don'ts

### Do
- 使用 figmaSans 搭配精準的可變字重（320–540）——顆粒化的字重控制就是設計
- 介面嚴格保持黑白——色彩只從產品內容來
- 所有互動元素使用藥丸（50px）與圓形（50%）幾何
- 套用 dashed 2px focus 外框——招牌的無障礙模式
- 在所有文字啟用 `"kern"` 功能
- 在標籤使用 figmaMono 大寫並加正字距
- 全站套用負字距（-0.1px 到 -1.72px）

### Don't
- 不要加介面顏色——單色盤面是絕對的
- 不要用標準字重（400、500、600、700）——使用可變字型的獨特停點（320、330、340、450、480、540）
- 不要在按鈕用銳角——只有藥丸與圓形幾何
- 不要用實線 focus 外框——虛線是招牌
- 不要把內文字重調高於 450——輕字重美學是核心
- 不要在內文用正字距——永遠是負字距

## 8. 響應式行為

### 斷點
| 名稱 | 寬度 | 主要變化 |
|------|------|----------|
| Small Mobile | <560px | 緊湊佈局、堆疊 |
| Tablet | 560–768px | 微調 |
| Small Desktop | 768–960px | 2 欄佈局 |
| Desktop | 960–1280px | 標準佈局 |
| Large Desktop | 1280–1440px | 展開 |
| Ultra-wide | 1440–1920px | 最大寬度 |

### 摺疊策略
- Hero 文字：86px → 64px → 48px
- 產品頁籤：手機水平捲動
- 功能區段：單欄堆疊
- 頁腳：多欄 → 堆疊

## 9. Agent Prompt 指南

### 快速色彩參考
- 一切：「純黑（`#000000`）」與「純白（`#ffffff`）」
- Glass Dark：「`rgba(0, 0, 0, 0.08)`」
- Glass Light：「`rgba(255, 255, 255, 0.16)`」

### 範例元件 Prompt
- 「在鮮豔多色漸層（綠、黃、紫、粉）上建立 hero。標題 86px figmaSans 字重 400、行高 1.0、字距 -1.72px。白色文字。白色藥丸 CTA 按鈕（50px 圓角、8px 18px padding）。」
- 「設計產品頁籤列，藥丸形按鈕（50px 圓角）。啟用：黑底、白字。未啟用：透明、黑字。figmaSans 20px 字重 480。」
- 「打造區段標籤：figmaMono 18px、大寫、字距 0.54px、黑色文字。啟用 kern。」
- 「建立內文 20px figmaSans 字重 330、行高 1.40、字距 -0.14px。白底純黑。」

### 迭代指南
1. 精準使用可變字型停點：320、330、340、450、480、540、700
2. 介面永遠是黑＋白——絕不在框架加色
3. 虛線 focus 外框，不是實線
4. 字距內文永遠負、mono 標籤永遠正
5. 按鈕／頁籤用藥丸（50px）、圖示按鈕用圓形（50%）
