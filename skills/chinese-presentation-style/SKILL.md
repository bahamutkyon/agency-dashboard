# 中文簡報美學規範

## 何時觸發此技能

當用戶請求「做投影片 / 做 PPT / 做簡報 / .pptx / 提案稿 / 工作報告」**且為中文場景**時,在呼叫 `mcp__powerpoint__*` 系列工具或寫 PPTXGenJS 腳本之前,**先讀本規範,讓產出符合中文簡報的視覺品味**。

英文簡報、純資料報表、學術 LaTeX 風格簡報**不適用本規範**。

---

## 核心原則

1. **少即是多** — 一張投影片一個主張,文案不超過 30 個漢字
2. **節奏感優先** — 簡報是時間藝術,留白比塞滿更重要
3. **不機翻外觀** — 預設模板有「ChatGPT 味」,必須主動覆蓋

---

## 字型規範

### 中文字型(必選其一,不混用)
| 字型 | 場景 | 備註 |
|---|---|---|
| **思源黑體 / Noto Sans CJK SC / TC** | 內文、資料報告、科技類 | 最通用、跨平台一致 |
| **蘋方 PingFang SC / TC** | macOS 環境、設計感 | Windows 沒有,**慎用** |
| **微軟正黑體 / Microsoft JhengHei** | Windows 環境 fallback | 不夠精緻但保險 |
| **冬青黑體 Hiragino Sans GB** | 文藝、品牌、人文類 | 質感最高 |
| **方正系列(蘭亭、悠黑、清刻本)** | 商務、政府、出版 | 需授權確認 |

❌ **禁用**:微軟雅黑(刺眼)、宋體 SimSun(太古板)、楷體(只能小範圍當引述)、新細明體(陳舊)

### 英文字型(中英混排時配對)
| 中文字型 | 配對英文 |
|---|---|
| 思源黑體 | Inter / Source Sans Pro / IBM Plex Sans |
| 蘋方 | SF Pro / Helvetica Neue |
| 微軟正黑 | Segoe UI / Arial |

❌ **禁用**:Times New Roman(嚴肅過頭)、Comic Sans(永遠不要)、Calibri(默認 ChatGPT 味)

### 字級
- **大標題**:48-72pt
- **小標**:28-36pt
- **內文**:20-24pt(**18pt 以下都太小**,後排看不見)
- **圖說 / 出處**:14-16pt

## 色彩規範

### 三色法則
**一張簡報最多用三個顏色族**(不算白與灰):
1. **主色**(品牌色 / 重點色)— 用在標題、強調、CTA
2. **輔色**(對比色)— 用在次要資訊、圖表的第二系列
3. **點綴色**(警示 / 補充)— 用在數字飆紅、關鍵數據

### 安全色盤(沒有品牌色時用這套)

| 用途 | 色號 | 說明 |
|---|---|---|
| 主色(科技藍) | `#2563EB` | 沉穩、專業 |
| 主色(自然綠) | `#059669` | 永續、健康 |
| 主色(品牌紅) | `#DC2626` | 衝擊、零售 |
| 文字主色 | `#111827` | 比純黑柔和 |
| 文字次色 | `#6B7280` | 圖說、標註 |
| 背景灰 | `#F3F4F6` | 區塊背景,別用純白 + 純黑 |
| 強調金 | `#D97706` | 數據聚焦點 |

❌ **禁用**:純黑 `#000000`(刺眼)、純紅 `#FF0000`(廉價)、漸層彩虹色(花俏)、PowerPoint 預設色盤(一眼看出沒設計)

### 對比度
- 文字與背景的對比度 ≥ **4.5:1**(WCAG AA)
- 深底淺字時,字色不要用純白,改 `#F9FAFB`(降低眩光)

## 留白與排版

### 安全邊距
- 簡報四邊**至少留 5%**(16:9 = 上下 38px、左右 67px)
- 標題與內文之間**留兩個小標字級的高度**(約 60-80px)
- 區塊之間用**留白**而不是分隔線

### 對齊
- **整份簡報統一靠左對齊**(不要置中,置中是 PPT 預設陷阱)
- 數字、價格、表格右對齊
- 圖片切齊文字基線,不要浮空

### 文案密度
| 投影片類型 | 字數上限 |
|---|---|
| 封面 | 標題 ≤ 15 字 + 副標 ≤ 25 字 |
| 章節分隔頁 | ≤ 8 字 |
| 內容頁 | 標題 ≤ 20 字 + 內文 ≤ 80 字(分點) |
| 數據頁 | 一個關鍵數字佔半屏 + 解讀 ≤ 30 字 |
| 結論頁 | 三個 bullet,每個 ≤ 15 字 |

**鐵則**:如果一張投影片字超過 100 字,**拆兩張**。聽眾不是來閱讀的,是來聽你講的。

## 結構節奏(一份完整簡報的骨架)

```
1. 封面          — 標題 + 副標 + 講者 + 日期
2. 為什麼是現在   — 問題 / 痛點 / 機會(1-2 張)
3. 我們的洞察    — 一句話總結你的觀點
4. 證據 1-3     — 數據 / 案例 / 對比
5. 解決方案     — 你提的方法
6. 怎麼落地     — 步驟 / 時程 / 資源
7. 預期成果     — 關鍵指標
8. 下一步       — CTA / 決策請求
9. 結尾頁       — 謝謝 + 聯絡方式
10. (備用) Q&A 與附錄
```

短簡報(≤ 10 張)壓縮成 1 / 2-4 / 5-6 / 7 / 8 五段。

## 特殊投影片類型

### 數據頁
- **一張投影片一個關鍵數字**,字級 100pt+,顏色用主色
- 旁邊放一句 ≤ 30 字的解讀(不要堆 5 個指標)
- 圖表只用必要的:長條(對比)、折線(趨勢)、圓餅(占比,類別 ≤ 5)
- ❌ 不用 3D、立體、漸層柱

### 引述頁
- 整段引言用 36pt+ 居中(此時可破靠左原則)
- 引言下方放出處 + 來源,字級小一倍、灰色
- 配一張高質量的人像 / 場景圖,**不要用 emoji**

### 對比頁(舊 vs 新 / 競品 vs 我們)
- 兩欄式,左右等寬
- 使用對比色,**不用紅綠**(色盲不友善,改藍橘 / 灰主色)
- 表頭加圖示(❌ vs ✅ 也行,但中文場景用「✗ / ✓」更內斂)

## 常見錯誤(AI 生 PPT 的標準翻車)

| 翻車 | 修正 |
|---|---|
| 每張塞 10 個 bullet | 拆成 3-5 張,每張 ≤ 3 個 bullet |
| 整篇黑底白字 | 黑底僅用於章節分隔頁,內容頁淺底 |
| 中英文沒空格 | 「打造AI產品」→「打造 AI 產品」|
| 用全形括號夾英文 | 「使用（GitHub）」→「使用 GitHub」|
| 引號不統一 | 中文用「」,不用 "" 或 ""|
| 數字用全形 | 「２０２６」→「2026」|
| 圖片不切齊 | 上下左右對齊文字 grid |
| Page X of Y 在右下 | 中文簡報用 「3 / 12」更乾淨 |
| 結尾「Thanks!」| 中文簡報結尾用「謝謝」「感謝聆聽」|
| 用 PPT 預設過場動畫 | 一律設成「淡出」或「無」|

## 與 PowerPoint MCP 的整合方式

呼叫 `mcp__powerpoint__create_presentation` / `mcp__powerpoint__add_slide` 時,**主動帶入本規範對應的參數**:

```python
# 範例:創建一張內容頁
mcp__powerpoint__add_slide(
    layout="title_content",
    title_text="<= 20 字標題>",
    title_font="Noto Sans CJK TC",
    title_size=36,
    title_color="#111827",
    content_text="<= 80 字,分 3 點>",
    content_font="Noto Sans CJK TC",
    content_size=22,
    content_color="#374151",
    background_color="#F3F4F6",
    margin_left=67,
    margin_right=67,
)
```

呼叫 `apply_professional_design` 時,**指定品牌色**:

```python
mcp__powerpoint__apply_professional_design(
    color_scheme={
        "primary": "#2563EB",
        "secondary": "#059669",
        "accent": "#D97706",
        "text": "#111827",
        "background": "#F9FAFB",
    },
    typography={
        "heading_cn": "Noto Sans CJK TC",
        "heading_en": "Inter",
        "body_cn": "Noto Sans CJK TC",
        "body_en": "Inter",
    }
)
```

## Sanity Check(產出後自我檢查)

生完 .pptx 後,**自己回答這 7 題**:
1. ☐ 每張投影片字數 ≤ 100?
2. ☐ 整份簡報 ≤ 3 個顏色族?
3. ☐ 字型只用 1-2 種?(中文 1 種 + 英文 1 種)
4. ☐ 中英文之間有空格?
5. ☐ 沒有純黑 / 純紅 / 漸層彩虹?
6. ☐ 有結構節奏(問題 → 洞察 → 證據 → 方案 → CTA)?
7. ☐ 結尾頁不是「Thanks!」是「謝謝」?

**任何一題答 No,改了再交付。**
