# 五維自評審 — 詳細評分標準與範例

本檔案是 `creative-quality-gate` skill 的閘門二完整評分指南。

---

## 五個維度詳細定義

### 1. Philosophy（哲學 / 觀點）

> 這個產出有沒有一個清晰的觀點 / 主張 / 立場？還是只是資訊堆疊？

| 分數 | 評斷標準 |
|---|---|
| **5 分** | 有明確主張,敢得罪一部分人,能被引用為「他說 X」 |
| **4 分** | 有觀點,方向清楚,但可以更直白 |
| **3 分** | 有觀點但太溫和,可以再尖銳一點 |
| **2 分** | 觀點模糊,讀完不確定他想說什麼 |
| **1 分** | 四平八穩的廢話,刪掉也不會有人發現 |

---

### 2. Hierarchy（層次 / 結構）

> 視覺層級和資訊層級是否一致？讀者眼睛知道先看哪裡嗎？

| 分數 | 評斷標準 |
|---|---|
| **5 分** | 3 秒內看到核心訊息,次要訊息退到背景,無認知負擔 |
| **4 分** | 主訊息清楚,但次要層級之間區隔可以更強 |
| **3 分** | 能看懂但需要從頭讀,沒有強引導 |
| **2 分** | 元素之間幾乎沒有區隔,看起來都一樣重要 |
| **1 分** | 所有元素等大、等色、等粗,像 Word 預設 |

---

### 3. Detail（細節 / 完成度）

> 標點、留白、對齊、顏色一致性、字距,是否到位？

| 分數 | 評斷標準 |
|---|---|
| **5 分** | 拿放大鏡找不出瑕疵,標點全形、字距協調、所有元素對齊在同一網格 |
| **4 分** | 整體完成度高,有 1-2 處細節可再打磨 |
| **3 分** | 大致一致但有少數錯漏（半全形混用、某一行對不齊） |
| **2 分** | 多處不一致,需要逐頁修正 |
| **1 分** | 明顯草稿味,連自己看都覺得不能交 |

---

### 4. Function（功能 / 任務達成）

> 這個產出有沒有達成它**該達成的事**？

| 分數 | 評斷標準 |
|---|---|
| **5 分** | 讀者讀完知道下一步要做什麼,或被打動到行動 |
| **4 分** | 任務達成,訊息清楚 |
| **3 分** | 傳達了資訊但沒激起反應 |
| **2 分** | 訊息不完整,讀者要猜測 |
| **1 分** | 讀完不知道目的是什麼 |

---

### 5. Innovation（創新 / 不可替代性）

> 換成任何 AI 工具來做,產出會一樣嗎？還是只有「你」會做成這樣？

| 分數 | 評斷標準 |
|---|---|
| **5 分** | 有獨特的視角、隱喻、結構、或細節,看得出創作者的判斷 |
| **4 分** | 有某些獨特之處,但整體仍可被類似工具復刻 |
| **3 分** | 執行到位但風格無辨識度 |
| **2 分** | 像範本套用,看不出創作者意圖 |
| **1 分** | 完全可被替換的 AI 罐頭 |

## 完整範例：從草稿到 emit

### 情境
使用者請你做一張「Q4 業績復盤」簡報封面。

### 草稿（閘門一檢查前）

> 標題：「Q4 業績復盤」
> 副標：「賦能團隊,沉澱方法論,打造更好的明天」
> 背景：紫色漸變
> Emoji：🚀 在標題前

### 閘門一檢查結果

對照 `references/anti-slop-blacklist.md`：

- ❌ 紫色漸變（A 類）→ 改為「黑底 + 1 個強調色」
- ❌ 副標通篇空話動詞（B 類）→ 改為具體數字
- ❌ 🚀 emoji（D 類）→ 拿掉
- ⚠️ 結構性主張缺失 → 副標應該說明「Q4 我們做對了什麼」而非口號

### 修正後（進入閘門二）

> 標題：「Q4：用 3 條產品線打回成長」
> 副標：「成交 +18% · 新客 320 · 客單 ¥4,200」
> 背景：黑底 + 主色橘 #FF5C28 在數字
> 字型：思源黑體 Bold 72pt / Regular 24pt

### 閘門二自評

```
## 自評
- Philosophy: 4/5 — 「3 條產品線打回成長」有觀點,但可以再尖銳（哪 3 條贏在哪？）
- Hierarchy:  5/5 — 標題→副標→數字三層權重明確
- Detail:     5/5 — 三個數字格式統一,前後綴對齊
- Function:   4/5 — 復盤主題明確,讀者理解 Q4 重點
- Innovation: 3/5 — 排版保險,但訊息夠強

最低分: 3 ≥ 3 ✅ 通過閘門
```

### 結論
✅ 全 ≥3,emit。

---

## 退回流程細節

若任一維度 <3：

1. **明確說出哪一維、為什麼**（不要含糊「我再改改」）
   - ✅ 好的：「Innovation 2/5 — 配色完全照模板,無辨識度」
   - ❌ 不好：「我覺得這個還可以再優化一下」

2. **針對該維度具體修正**,不要全部重做
   - 範例：Detail 2/5 因字距 → 只改字距,其他維度保留

3. **重評該維度**,其他維度若沒動可以保留分數

4. 全部 ≥3 才能 emit

---

## 常見虛高分警訊

如果你的自評**全部 5/5**,大概率沒認真檢查。常見偷懶模式：

- **過度肯定**：每維度都寫「完美」「完整」「到位」→ 八成沒真的對照標準
- **無修正紀錄**：第一次評就全 5,沒寫「已修正 X」→ 不可能一次到位
- **抽象用語**：「整體不錯」「品質良好」→ 沒對應到具體維度

健康的自評通常會有 1-2 個 3 分,1 個 4 分,1-2 個 5 分。全 5 不是優秀,是粗心。
