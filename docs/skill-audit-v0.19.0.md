## Sample reasoning

### `verification-before-completion`
- **academic-anthropologist**: 宣称文化系统完成前,验证其内部一致性、功能合理性和原始元素的正确理解
- **academic-geographer**: 地理学家有严格的科学规则约束,每项设计完成前必须验证物理一致性和遵守地理学原理
- **academic-historian**: 歷史學家的核心使命就是驗證歷史一致性；輸出任何時期真實性報告前必須檢驗資料、識別時代錯誤、確認設定的學術可信度

### `writing-plans`
- **academic-anthropologist**: 制定详细的文化系统设计规划,确保每个元素(亲属制、交换系统、仪式)的功能和相互依赖关系
- **academic-geographer**: 地形→气候→水文→生物群落→人文地理,这是有序的多步骤设计流程,需要前期规划避免返工
- **academic-narratologist**: 故事改進是多步驟任務（結構調整→人物弧線修正→敘事欠債補償），需要制定清晰的改進路線圖

### `brainstorming`
- **academic-anthropologist**: 设计前必须探索用户需求、文化约束和设计空间,避免仓促拼湊文化陈词滥调
- **academic-geographer**: 构建地理一致的世界系统前,需要深入探索气候驱动、资源约束、聚落逻辑等复杂的地理相互作用
- **academic-narratologist**: 敘事學家需要幫客戶在提出分析前先探索故事理念、人物設定、情節方向——這是創意基礎

### `systematic-debugging`
- **academic-anthropologist**: 识别和修复文化系统中的矛盾、逻辑断裂和不一致性——这是评估文化真实性的核心工作
- **academic-geographer**: 当地理设计出现矛盾(如河流分流、气候不合理),需要系统性地诊断问题根源并修复一致性
- **academic-historian**: 用系統性方法論診斷歷史敘述中的矛盾、技術/經濟/社會不一致之處，就像除錯複雜系統一樣找根本原因而非表面症狀

### `test-driven-development`
- **academic-historian**: 將歷史驗證標準化：先定義「時期X應符合的物質文化/社會結構標準」，再逐一驗證設定中的主張是否通過這些標準
- **accounts-payable-agent**: 複雜的業務邏輯（冪等性、發票驗證、限額控制、重複支付檢查）必須通過 TDD 確保正確性
- **agentic-identity-trust**: 身份系統涉及密鑰管理、簽名驗證、授權鏈驗證等高風險代碼，TDD 對確保正確性和安全性最關鍵

- **agentic-identity-trust**: 規範文檔確保其他 agents 能正確和安全地集成身份系統，清晰的密碼學、授權邏輯說明直接影響整體安全性
